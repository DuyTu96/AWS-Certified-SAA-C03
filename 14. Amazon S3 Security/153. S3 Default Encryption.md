## Nội dung chính: Default Encryption vs Bucket Policies trong Amazon S3

Bài học này giải thích sự khác nhau giữa **Default Encryption** và **Bucket Policy** khi muốn bắt buộc mã hóa object trong **Amazon S3**.

---

## 1. Default Encryption trong Amazon S3

Hiện tại, mặc định tất cả S3 bucket mới đều có:

```text
Default encryption = SSE-S3
```

Điều này có nghĩa là:

* Object mới upload vào bucket sẽ tự động được mã hóa.
* Cơ chế mặc định là **SSE-S3**.
* **SSE-S3** sử dụng **Amazon S3 managed keys**.
* Người dùng không cần cấu hình thêm nếu chỉ muốn mã hóa cơ bản.

Tuy nhiên, bạn có thể thay đổi **default encryption** sang cơ chế khác, ví dụ:

```text
SSE-KMS
```

---

## 2. Bucket Policy có thể dùng để force encryption

Ngoài **Default Encryption**, bạn cũng có thể dùng **Bucket Policy** để bắt buộc request upload object phải có encryption header phù hợp.

Ví dụ:

* Nếu user thực hiện **PUT Object** nhưng không có header yêu cầu mã hóa bằng **SSE-KMS**, request sẽ bị deny.
* Nếu user upload object nhưng không có header cho **SSE-C**, request cũng có thể bị deny.

Nói cách khác:

> **Bucket Policy** có thể từ chối các API call không đáp ứng đúng yêu cầu encryption.

---

## 3. Ví dụ logic Bucket Policy

### Force SSE-KMS

Bucket Policy có thể kiểm tra request upload object.

Nếu request không có encryption header:

```text
x-amz-server-side-encryption: aws:kms
```

thì deny request.

Ý nghĩa:

```text
PUT Object without SSE-KMS header => Deny
```

---

### Force SSE-C

Với **SSE-C**, policy có thể kiểm tra request có khai báo **customer side algorithm** hay không.

Nếu không có header liên quan đến **SSE-C**, request sẽ bị deny.

Ý nghĩa:

```text
PUT Object without SSE-C header => Deny
```

---

## 4. Bucket Policy được evaluate trước Default Encryption

Điểm rất quan trọng:

> **Bucket Policies are evaluated before Default Encryption settings.**

Điều này nghĩa là:

* Khi một request upload object vào S3 bucket, **Bucket Policy** sẽ được kiểm tra trước.
* Nếu Bucket Policy deny request, object sẽ không được upload.
* Default Encryption chỉ áp dụng sau khi request đã được policy cho phép.

Ví dụ:

Bucket có **Default Encryption = SSE-S3**, nhưng Bucket Policy yêu cầu **SSE-KMS**.

Nếu user upload object mà không có header **SSE-KMS**, request sẽ bị deny trước, dù bucket có default encryption.

---

## 5. So sánh Default Encryption và Bucket Policy

| Cơ chế                                  | Vai trò                                              |
| --------------------------------------- | ---------------------------------------------------- |
| **Default Encryption**                  | Tự động mã hóa object nếu request được chấp nhận     |
| **Bucket Policy**                       | Kiểm soát và deny request nếu không đúng rule        |
| **SSE-S3 default**                      | Được bật mặc định cho bucket mới                     |
| **SSE-KMS default**                     | Có thể cấu hình thay cho SSE-S3                      |
| **Force encryption bằng Bucket Policy** | Bắt buộc request phải có encryption header mong muốn |

---

## Tổng kết nhanh

* Amazon S3 hiện có **Default Encryption = SSE-S3** mặc định.
* Có thể đổi default encryption sang **SSE-KMS**.
* **Bucket Policy** có thể dùng để force encryption.
* Bucket Policy có thể deny **PUT Object** nếu thiếu encryption header.
* Có thể force các cơ chế như:

  * **SSE-KMS**
  * **SSE-C**
* **Bucket Policy** luôn được evaluate trước **Default Encryption settings**.

## Ghi nhớ cho exam

```text
Default Encryption encrypts automatically.
Bucket Policy enforces encryption requirements.
Bucket Policy is evaluated before Default Encryption.
```

Nói ngắn gọn:

> **Default Encryption** giúp tự động mã hóa object, còn **Bucket Policy** giúp bắt buộc request phải tuân thủ đúng loại encryption mong muốn.
