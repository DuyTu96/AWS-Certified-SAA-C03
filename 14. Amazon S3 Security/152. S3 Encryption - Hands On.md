## Nội dung chính: Thực hành Encryption trong Amazon S3

Bài học này thực hành cấu hình **encryption** cho object trong **Amazon S3** thông qua AWS Console.

---

## 1. Tạo S3 Bucket với Default Encryption

Giảng viên tạo một bucket tên:

```text
demo-encryption-stephane-v2
```

Khi tạo bucket:

* Giữ nguyên một số cấu hình mặc định.
* Bật **Bucket Versioning**.
* Cấu hình **Default Encryption**.

Trong phần **Default Encryption**, Amazon S3 Console hiện có 3 lựa chọn chính:

* **SSE-S3**
* **SSE-KMS**
* **DSSE-KMS**

Ở bước đầu, giảng viên chọn **SSE-S3** làm default encryption.

Sau khi tạo bucket, bucket đã được bật **default encryption**.

---

## 2. Upload object và kiểm tra SSE-S3

Giảng viên upload file:

```text
coffee.jpg
```

Sau khi upload, mở file và kiểm tra phần:

```text
Server-side encryption settings
```

Kết quả cho thấy object đã được mã hóa bằng:

```text
Server-side encryption with Amazon S3 managed keys
```

Tức là:

```text
SSE-S3
```

Điều này chứng minh rằng khi bucket có **default encryption = SSE-S3**, object upload lên sẽ tự động được mã hóa bằng **SSE-S3**.

---

## 3. Thay đổi encryption của một object

Amazon S3 cho phép chỉnh sửa encryption mechanism của từng object.

Khi click **Edit** trong phần encryption setting của object:

* Nếu thay đổi **server-side encryption**, S3 sẽ tạo một **new version** của object.
* Đây là lý do bài học bật **Bucket Versioning** trước đó.

Giảng viên thay đổi encryption cho riêng object `coffee.jpg` bằng cách:

* Override bucket default encryption.
* Chọn **SSE-KMS** thay vì **SSE-S3**.

---

## 4. Chọn KMS Key cho SSE-KMS

Khi dùng **SSE-KMS**, cần chỉ định một **KMS key**.

Có 2 cách:

* Nhập trực tiếp **KMS key ARN**.
* Chọn từ danh sách **KMS keys** có sẵn.

Trong bài học, key mặc định có sẵn là:

```text
aws/s3
```

Đây là **default KMS key for the S3 service**.

Điểm quan trọng:

* Dùng default key `aws/s3` sẽ không tốn thêm phí tạo key hàng tháng.
* Nếu tự tạo **Customer managed KMS key**, key đó sẽ xuất hiện trong danh sách.
* Nhưng **Customer managed KMS key** có thể phát sinh chi phí hàng tháng.

Sau khi save changes, object `coffee.jpg` có thêm một version mới.

Version hiện tại đã được mã hóa bằng:

```text
SSE-KMS
```

với key:

```text
aws/s3
```

---

## 5. Versioning khi thay đổi encryption

Vì bucket đã bật **Bucket Versioning**, nên sau khi đổi encryption của object:

* Object có version cũ dùng **SSE-S3**.
* Object có version mới dùng **SSE-KMS**.

Điểm cần nhớ:

> Khi edit server-side encryption của object, Amazon S3 tạo ra một **new version** của object với encryption setting mới.

---

## 6. Chọn encryption khi upload object mới

Giảng viên tiếp tục upload file khác, ví dụ:

```text
beach.jpg
```

Trong quá trình upload, tại phần **Properties**, có thể cấu hình:

```text
Server-side encryption
```

Tại đây có thể chọn:

* Dùng **default encryption mechanism** của bucket.
* Hoặc override bằng:

  * **SSE-S3**
  * **SSE-KMS**
  * **DSSE-KMS**

Đây là cách cấu hình encryption trực tiếp khi upload từng object.

---

## 7. Thay đổi Default Encryption của bucket

Trong phần **Properties** của bucket, có mục:

```text
Default encryption
```

Khi edit, có thể chọn default encryption là:

* **SSE-S3**
* **SSE-KMS**
* **DSSE-KMS**

Nếu chọn **SSE-KMS**, sẽ có thêm option:

```text
Bucket Key
```

---

## 8. Bucket Key trong SSE-KMS

Khi dùng **SSE-KMS**, option **Bucket Key** có thể được bật.

Mục đích:

> Giảm chi phí bằng cách giảm số lượng API calls tới **AWS KMS**.

Trong bài học:

* **Bucket Key** được bật mặc định.
* Nếu dùng **SSE-S3**, setting này không có tác dụng.

Điểm cần nhớ:

```text
S3 Bucket Key = reduce AWS KMS API calls = reduce cost
```

---

## 9. DSSE-KMS là gì?

Trong Console có option:

```text
DSSE-KMS
```

Giảng viên không đi sâu vào phần này.

Ý chính:

```text
DSSE-KMS = two levels of encryption on KMS
```

Có thể hiểu là:

> **DSSE-KMS** là cơ chế mã hóa mạnh hơn dựa trên **AWS KMS**, dùng **double encryption** hoặc **dual-layer encryption**.

---

## 10. Vì sao không thấy SSE-C trong Console?

Trong AWS Console, chỉ thấy:

* **SSE-S3**
* **SSE-KMS**
* **DSSE-KMS**

Không thấy:

```text
SSE-C
```

Lý do:

> **SSE-C** chỉ có thể thực hiện qua **CLI**, không cấu hình được trực tiếp trong AWS Console.

Vì vậy, không thể enable **SSE-C** trong phần Default Encryption của bucket trên Console.

---

## 11. Client-side Encryption không cần cấu hình trên AWS Console

Với **Client-side encryption**:

* Client tự encrypt data trước khi upload lên S3.
* Client tự decrypt data sau khi download từ S3.
* AWS S3 chỉ lưu dữ liệu đã được mã hóa.

Vì vậy:

> Không cần báo cho AWS biết object đang dùng **client-side encryption**.

Do đó, **client-side encryption** không xuất hiện như một option trong AWS Console.

---

## Tổng kết nhanh

| Nội dung                   | Ý nghĩa                                                    |
| -------------------------- | ---------------------------------------------------------- |
| **Default Encryption**     | Encryption mặc định cho object upload vào bucket           |
| **SSE-S3**                 | Server-side encryption với Amazon S3 managed keys          |
| **SSE-KMS**                | Server-side encryption với AWS KMS key                     |
| **DSSE-KMS**               | Double encryption / dual-layer encryption dựa trên AWS KMS |
| **Bucket Versioning**      | Khi đổi encryption của object sẽ tạo new version           |
| **aws/s3**                 | Default KMS key for S3 service                             |
| **Bucket Key**             | Giảm AWS KMS API calls, giúp giảm cost                     |
| **SSE-C**                  | Không cấu hình được trên Console, chỉ dùng qua CLI         |
| **Client-side encryption** | Client tự encrypt/decrypt, AWS không cần biết              |

## Ghi nhớ cho exam

* Trong Console chỉ cấu hình được **SSE-S3**, **SSE-KMS**, **DSSE-KMS**.
* **SSE-C** không có trong Console, phải dùng **CLI**.
* **Client-side encryption** không cần setting trên AWS vì dữ liệu đã được encrypt trước khi upload.
* Thay đổi encryption của object sẽ tạo **new version** nếu bucket bật **Versioning**.
* **S3 Bucket Key** dùng với **SSE-KMS** để giảm số lần gọi **AWS KMS** và giảm chi phí.
