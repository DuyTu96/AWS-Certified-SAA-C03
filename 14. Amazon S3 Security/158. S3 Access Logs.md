# 158. S3 Access Logs

## 📝 S3 Access Logs – Ghi lại toàn bộ truy cập vào S3 Bucket

### 1. **S3 Access Logs là gì?**

* **S3 Access Logs** cho phép ghi lại toàn bộ các request được gửi đến **S3 Bucket** nhằm phục vụ mục đích **audit** và theo dõi hoạt động.
* Mọi request đến bucket đều có thể được ghi log:

  * ✅ Request được **authorized**.
  * ✅ Request bị **denied**.
  * ✅ Request đến từ **bất kỳ AWS Account nào**.

---

### 2. **Cách hoạt động của S3 Access Logs**

* Khi bật **S3 Access Logs** cho một bucket:

  * Các request gửi đến bucket sẽ được ghi lại.
  * Các log này được lưu thành **file** trong một **target logging bucket** khác.
* Các file log sau đó có thể được phân tích bằng các công cụ như:

  * **Amazon Athena**.

> Luồng hoạt động:
>
> `Client Request → S3 Bucket → S3 Access Logs → Logging Bucket → Amazon Athena (phân tích dữ liệu)`

---

### 3. **Yêu cầu đối với Logging Bucket**

* **Target logging bucket** phải nằm trong **cùng AWS Region** với bucket đang được theo dõi.
* Đây là yêu cầu khi cấu hình **S3 Access Logs**.

---

### 4. **Định dạng của Access Logs**

* Các file log được lưu theo một **format xác định** do AWS quy định.
* [Có thể tham khảo tài liệu chính thức của AWS để xem chi tiết cấu trúc của log format khi cần phân tích dữ liệu.](https://docs.aws.amazon.com/AmazonS3/latest/userguide/LogFormat.html)

---

### 5. ⚠️ **Không được dùng chính bucket đang theo dõi làm Logging Bucket**

* **Tuyệt đối không** cấu hình **logging bucket** trùng với **bucket đang được monitor**.
* Nếu làm vậy sẽ xảy ra **logging loop**:

  * Một thao tác ghi object sinh ra một log.
  * Việc ghi log lại tiếp tục tạo thêm log.
  * Quá trình này lặp đi lặp lại vô hạn (**infinite loop**).

Hậu quả:

* 📈 Số lượng object trong bucket tăng theo cấp số nhân.
* 💸 Chi phí lưu trữ và vận hành AWS tăng rất lớn.
* 🚫 Đây là cấu hình cần tránh tuyệt đối.

---

### 6. 📌 **Kết luận**

* **S3 Access Logs** giúp ghi lại toàn bộ hoạt động truy cập vào **S3 Bucket** để phục vụ **audit** và phân tích.
* Log được lưu trong một **target logging bucket** và có thể phân tích bằng **Amazon Athena**.
* Logging bucket phải nằm trong **cùng AWS Region**.
* ⚠️ **Không bao giờ cấu hình logging bucket là chính bucket đang được theo dõi**, vì sẽ gây ra **logging loop** và phát sinh chi phí rất lớn.
