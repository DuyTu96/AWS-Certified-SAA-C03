# 290. Lambda Function Performance

## 🎯 Giới thiệu
Bài này nói về các cách tối ưu **Lambda function performance** thông qua:
- **RAM**
- **Timeout**
- **Execution context**
- **/tmp storage**

Mục tiêu là hiểu các điểm rất hay xuất hiện trong câu hỏi thi AWS.

## 1. RAM và vCPU trong Lambda ⚙️
- Lambda hiện dùng **128 MB RAM** trong ví dụ, nhưng có thể tăng lên tới **10 GB RAM**.
- RAM tăng theo bước **1 MB increments**.
- **Không set vCPU trực tiếp** trong Lambda.
- Muốn có thêm **vCPU** thì phải tăng **RAM**.
- Khi đạt **1,792 MB RAM**, Lambda có tương đương **1 full vCPU**.
- Sau mức này, function có thể có **nhiều hơn 1 vCPU**.
- Nếu ứng dụng **CPU-bound**:
  - nghĩa là có nhiều tính toán
  - hãy tăng **RAM** để cải thiện hiệu năng và giảm thời gian chạy
- Nếu đã có nhiều vCPU hơn 1, cần **multi-threading** để tận dụng thêm sức mạnh xử lý.

## 2. Timeout của Lambda ⏱️
- **Default timeout** của Lambda là **3 seconds**.
- Nếu chạy quá 3 giây, function sẽ bị **timeout error**.
- Có thể tăng timeout tối đa tới **900 seconds = 15 minutes**.
- Lambda phù hợp cho các tác vụ có thời gian thực thi từ **0 giây đến 15 phút**.
- Nếu lâu hơn **15 phút** thì Lambda không phù hợp; transcript gợi ý dùng:
  - **Fargate**
  - **ECS**
  - **EC2**

## 3. Execution Context và /tmp ♻️
- Lambda có **execution context**:
  - là môi trường runtime tạm thời
  - dùng để khởi tạo các dependency bên ngoài của code Lambda
- Nên khởi tạo ở đây:
  - **database connections**
  - **HTTP clients**
  - **SDK clients**
- Execution context có thể được **giữ lại một thời gian** để chờ lần invoke tiếp theo.
- Vì vậy, nếu Lambda bị gọi nhiều lần liên tiếp:
  - context cũ có thể được **reuse**
  - các connection/clients có thể được tái sử dụng
  - hiệu năng sẽ tốt hơn

### Ví dụ best practice
- **Sai**: tạo `database.connect` bên trong handler mỗi lần invoke
- **Đúng**: khởi tạo connection **bên ngoài handler**
  - chỉ làm một lần
  - reuse cho các lần gọi sau
- Best practice chung:
  - mọi thứ **mất nhiều thời gian để khởi tạo**
  - hãy đặt **ngoài function handler**
  - để tái sử dụng qua nhiều executions

### /tmp storage
- Execution context bao gồm cả thư mục **`/tmp`**
- Đây là nơi có thể ghi file tạm và dùng lại giữa các executions
- Lambda có **10 GB disk space** cho mục đích này
- Có thể lưu file tạm để:
  - xử lý file lớn
  - thực hiện các thao tác cần disk space
- `/tmp` có thể giữ lại dữ liệu để lần invoke sau đọc lại cùng file cũ
- Transcript nói có thể ghi các file rất nặng, **up to half a gigabyte**, vào `/tmp`

### Lưu ý về persistence và encryption
- Nếu cần **persistent storage** không phải tạm thời:
  - hãy lưu vào nơi như **Amazon S3**
- Nếu muốn **encrypt content** trên `/tmp`:
  - không có setting trực tiếp trong Lambda
  - phải dùng **KMS**
  - tạo **data keys**
  - rồi tự encrypt dữ liệu trong temp space bằng các data keys đó

## 📊 Bảng tóm tắt
| Tiêu chí | Mô tả |
|----------|------|
| RAM | Tăng RAM để tăng vCPU ngầm định và cải thiện hiệu năng, đặc biệt với workload CPU-bound |
| vCPU | Không set trực tiếp; đạt **1,792 MB RAM** thì tương đương **1 vCPU** |
| Timeout | Default **3 seconds**, tối đa **900 seconds (15 minutes)** |
| Use case phù hợp | Task chạy trong khoảng **0 đến 15 phút** |
| Execution context | Môi trường tạm thời giúp reuse database connections, HTTP clients, SDK clients |

| Persistent data | Dữ liệu không tạm thời nên lưu vào **S3** |
| Encrypt `/tmp` | Dùng **KMS** và data keys, không có setting trực tiếp trong Lambda |

## 💡 Mẹo ghi nhớ cho kỳ thi AWS
- **RAM tăng = vCPU tăng ngầm định**: không set vCPU trực tiếp.
- Mốc rất quan trọng: **1,792 MB = 1 vCPU**.
- **Lambda timeout max = 15 minutes**.
- **Connection/clients nên đặt ngoài handler** để reuse.
- **`/tmp` là storage tạm**, không phải nơi lưu bền vững.
- Muốn lưu lâu dài thì nhớ đến **S3**.
- Muốn mã hóa dữ liệu trong `/tmp` thì nhớ **KMS**.

## ✅ Kết luận
Lambda performance trong bài này xoay quanh 4 ý chính:
- tăng **RAM** để tăng khả năng xử lý,
- điều chỉnh **timeout** trong giới hạn 15 phút,
- tận dụng **execution context** để reuse các kết nối và client,
- dùng **/tmp** cho dữ liệu tạm thời, còn dữ liệu bền vững thì đưa lên **S3**.
