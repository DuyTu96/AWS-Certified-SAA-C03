# 283. Lambda Environment Variables

## 🎯 Giới thiệu
Lambda Environment Variables là các cặp `key-value` dạng chuỗi, dùng để thay đổi hành vi của function mà không cần sửa code. Đây là một cấu hình phổ biến trong lập trình và cũng được hỗ trợ trong Lambda.

## 1. Khái niệm chính
- Environment Variables trong Lambda là các giá trị theo dạng `string`.
- Mục đích: điều chỉnh behavior của function mà không phải cập nhật source code.
- Các biến này sẽ có sẵn cho code của bạn khi Lambda chạy.

## 2. Cách Lambda xử lý environment variables
- Code của function có thể truy cập các environment variables đã cấu hình.
- Lambda Service cũng tự thêm các system environment variables riêng của nó.
- Điều này giúp Lambda hoạt động giống mô hình dùng environment variables quen thuộc trong lập trình.

## 3. Bảo mật với KMS
- Lambda cho phép encrypt environment variables.
- Có thể dùng `KMS` để lưu trữ các giá trị secret.
- Secret có thể được encrypt bằng:
  - `Lambda service key`
  - hoặc `customer master key` của bạn

## 📊 Bảng tóm tắt
| Tiêu chí | Mô tả |
|----------|------|
| Dạng dữ liệu | `Key-value` dạng chuỗi |
| Mục đích | Thay đổi behavior của Lambda mà không sửa code |
| Phạm vi sử dụng | Có sẵn cho code của function |
| Hệ thống hỗ trợ | Lambda Service cũng có system environment variables |
| Bảo mật | Có thể encrypt bằng `KMS` |
| Key mã hóa | `Lambda service key` hoặc `customer master key` |

## 💡 Mẹo ghi nhớ cho kỳ thi AWS
- Nhớ rằng Environment Variables trong Lambda dùng để **tách cấu hình khỏi code**.
- Nếu đề bài nói đến **secret values** trong environment variables, hãy nghĩ ngay đến `KMS`.
- Câu hỏi thi thường xoay quanh ý: **không cần deploy lại code vẫn đổi được behavior**.

## ✅ Kết luận
Lambda Environment Variables là cách đơn giản để truyền cấu hình cho function dưới dạng chuỗi `key-value`, giúp thay đổi hành vi mà không sửa code. Chúng cũng có thể được encrypt bằng `KMS` để bảo vệ các giá trị nhạy cảm.
