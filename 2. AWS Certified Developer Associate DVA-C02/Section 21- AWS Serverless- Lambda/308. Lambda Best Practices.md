# 308. Lambda Best Practices

## 🎯 Giới thiệu
Bài này tổng hợp các **Lambda Best Practices** quan trọng để tối ưu hiệu năng, giảm chi phí và tránh lỗi phổ biến khi đi thi AWS.

## 1. Tối ưu `function handler`
- Làm các tác vụ nặng **bên ngoài `function handler`** để giảm thời gian chạy của handler.
- Nên khởi tạo:
  - Kết nối database
  - AWS SDK
  - Dependencies hoặc datasets
  - ...ở **ngoài handler**
- Mục tiêu: handler chạy càng ngắn càng tốt.

## 2. Dùng `environment variables` đúng cách
- Dùng `environment variables` cho các giá trị có thể thay đổi theo thời gian:
  - Database connection strings
  - `S3 bucket name`
  - Các giá trị cấu hình khác
- **Không** hardcode các giá trị này trong code.
- Với password và dữ liệu nhạy cảm, có thể **encrypt environment variables bằng KMS**.

## 3. Giảm kích thước deployment và tránh lỗi thiết kế
- Giữ **deployment package** nhỏ, chỉ chứa những gì cần cho runtime.
- Nếu function quá lớn, hãy **chia nhỏ** ra.
- Luôn nhớ các **Lambda limits** liên quan đến package size.
- Nếu cần dùng lại thư viện, cân nhắc dùng **Lambda layers**.
- **Không viết recursive Lambda**:
  - Lambda function **không nên tự gọi chính nó**
  - Việc này có thể gây tốn kém và trở thành disaster.

## 📊 Bảng tóm tắt
| Tiêu chí | Mô tả |
|----------|------|
| Handler | Làm việc nặng ngoài `function handler` để giảm thời gian chạy |
| Cấu hình | Dùng `environment variables` cho giá trị thay đổi |
| Bảo mật | Encrypt biến môi trường nhạy cảm bằng `KMS` |
| Deployment | Giảm kích thước package, chỉ giữ runtime necessities |
| Tái sử dụng | Dùng `Lambda layers` cho thư viện dùng chung |
| Thiết kế | Tránh recursive Lambda, không để function gọi lại chính nó |

## 💡 Mẹo ghi nhớ cho kỳ thi AWS
- **Handler nhẹ - Env linh hoạt - Package nhỏ - Không recurse**
- Gặp câu hỏi về tối ưu Lambda, ưu tiên nhớ 4 ý:
  - Khởi tạo ngoài handler
  - Dùng `environment variables`
  - Encrypt bằng `KMS` nếu là sensitive data
  - Dùng `Lambda layers` khi cần reuse libraries

## ✅ Kết luận
Các best practices của `Lambda` tập trung vào 3 mục tiêu chính: **giảm thời gian chạy**, **quản lý cấu hình an toàn**, và **giữ deployment gọn nhẹ**. Đi thi AWS, đặc biệt lưu ý: **không hardcode config**, **dùng KMS cho secrets**, và **không để Lambda tự gọi chính nó**.
