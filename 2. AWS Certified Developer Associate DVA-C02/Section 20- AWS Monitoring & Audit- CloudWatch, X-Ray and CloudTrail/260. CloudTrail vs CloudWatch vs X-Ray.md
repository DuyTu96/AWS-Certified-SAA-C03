# 260. CloudTrail vs CloudWatch vs X-Ray

## 🎯 Giới thiệu
- Ba dịch vụ này dễ nhầm lẫn vì đều liên quan đến theo dõi hệ thống, nhưng mục đích khác nhau:
  - `CloudTrail` dùng cho **audit API calls**
  - `CloudWatch` dùng cho **monitoring**
  - `X-Ray` dùng cho **trace analysis** và quan sát hệ thống phân tán

## 1. CloudTrail
- Dùng để **audit API calls** được thực hiện trong account.
- Các API calls này có thể đến từ:
  - user
  - services
  - AWS Console
- Hữu ích khi:
  - muốn phát hiện **unauthorized calls**
  - muốn tìm **root cause** của thay đổi do API calls gây ra

## 2. CloudWatch
- Dùng cho **monitoring**.
- Thành phần chính được nhắc đến trong transcript:
  - `CloudWatch Metrics` để theo dõi metrics
  - `CloudWatch Logs` để lưu application logs
  - `CloudWatch Alarms` để gửi notification khi metrics bất thường
- Tóm lại, `CloudWatch` xoay quanh việc giám sát hệ thống và ứng dụng.

## 3. X-Ray
- Dùng cho **automated trace analysis**.
- Có **central service map visualization**, rất hữu ích với **distributed service**.
- Hỗ trợ:
  - debugging
  - xem `latency`
  - xem `errors`
  - `fault analysis`
  - `request tracking` across distributed systems
- Trong transcript, `X-Ray` được mô tả là **granular hơn** và **trace-oriented** hơn so với `CloudWatch`.

## 📊 Bảng tóm tắt
| Tiêu chí | Mô tả |
|----------|------|
| `CloudTrail` | Audit `API calls` trong account, hữu ích để phát hiện unauthorized calls và tìm root cause của thay đổi |
| `CloudWatch` | Monitoring bằng `Metrics`, `Logs`, và `Alarms` |
| `X-Ray` | `Trace analysis`, service map, debugging, theo dõi latency, errors và faults trong hệ thống phân tán |
| Mức độ | `CloudWatch` là tổng quan, `X-Ray` chi tiết hơn, `CloudTrail` tập trung vào audit |
| Trọng tâm | `CloudTrail` = API calls, `CloudWatch` = monitoring, `X-Ray` = tracing |

## 💡 Mẹo ghi nhớ cho kỳ thi AWS
- `CloudTrail` = **ai đã gọi API gì?**
- `CloudWatch` = **hệ thống đang hoạt động thế nào?**
- `X-Ray` = **request đi qua hệ thống phân tán ra sao?**
- Nếu câu hỏi nói về:
  - **audit** hoặc **API calls** → nghĩ ngay `CloudTrail`
  - **metrics / logs / alarms / monitoring** → nghĩ `CloudWatch`
  - **trace / distributed system / latency / debugging** → nghĩ `X-Ray`

## ✅ Kết luận
- `CloudTrail` dùng để audit `API calls`.
- `CloudWatch` dùng để monitoring bằng `Metrics`, `Logs`, `Alarms`.
- `X-Ray` dùng để trace analysis và debug hệ thống phân tán.
- Đi thi AWS, chỉ cần nhớ 3 từ khóa chính: **audit - monitoring - tracing**.
