# 50. AMI Hands On

## 🎯 Giới thiệu
Bài thực hành minh họa cách tạo **custom AMI** từ EC2 instance đã cài sẵn **HTTPD / Apache web server**, sau đó launch instance mới từ AMI để boot nhanh hơn.

## 1. Launch EC2 instance ban đầu 🚀

Trong demo, tạo EC2 instance với:

- AMI: **Amazon Linux 2**.
- Instance type: **t2.micro**.
- Key pair: chọn key pair đã có.
- Security group: dùng security group có sẵn, ví dụ **launch-wizard-1**.
- Storage: giữ mặc định.

## 2. Dùng EC2 User Data để cài HTTPD 📜

Trong phần **Advanced details**, thêm **User Data**.

Điểm quan trọng:

- Copy script để install **HTTPD** / **Apache web server**.
- Không copy dòng cuối tạo index file.
- Mục tiêu là tạo instance có Apache được cài sẵn, sau đó package thành AMI.

Sau khi launch:

- Instance cần một chút thời gian để chạy **EC2 User Data script**.
- Khi truy cập bằng **HTTP protocol**, sẽ thấy test page của Apache web server.

```mermaid
flowchart LR
    A[Launch EC2 Amazon Linux 2] --> B[Run EC2 User Data]
    B --> C[Install HTTPD]
    C --> D[Apache test page]
```

## 3. Create AMI từ EC2 instance 📦

Khi web server đã sẵn sàng:

- Right-click EC2 instance.
- Chọn **Image and templates**.
- Chọn **Create image**.
- Đặt tên, ví dụ **demo image**.
- Click **Create image**.

Trong menu **AMIs**:

- AMI mới xuất hiện.
- Ban đầu status là **pending**.
- Sau đó chuyển sang trạng thái created/available.

## 4. Launch instance từ custom AMI 🔁

Có thể launch instance trực tiếp từ AMI mới:

- Vào tab **My AMIs**.
- Chọn **demo image**.
- Chọn key pair hoặc không.
- Chọn security group có sẵn.
- Thêm **User Data** mới chỉ để tạo file index.

📌 Không cần reinstall HTTPD vì HTTPD đã nằm trong custom AMI.

```mermaid
flowchart LR
    A[Custom AMI có HTTPD] --> B[Launch EC2 mới]
    B --> C[Run User Data tạo index file]
    C --> D[Hello World page]
```

## 5. Lợi ích của AMI trong demo ✅

Instance launch từ custom AMI nhanh hơn vì:

- Không cần cài lại HTTPD.
- Software đã được package sẵn trong AMI.
- Chỉ cần customization nhỏ ở cuối bằng User Data.

Use case mở rộng trong bài:

- Có thể package security software.
- Có thể package prerequisite software.
- Giúp boot-up nhanh hơn.

## 6. Cleanup 🧹

Sau demo:

- Terminate hai EC2 instances.

## 📊 Bảng tóm tắt

| Giai đoạn | Nội dung |
|----------|----------|
| Launch EC2 ban đầu | Amazon Linux 2, t2.micro |
| User Data lần đầu | Install HTTPD / Apache web server |
| Create AMI | Tạo custom AMI từ EC2 instance |
| Launch từ AMI | Dùng My AMIs để chọn demo image |
| User Data lần hai | Chỉ tạo index file, không install HTTPD |
| Lợi ích | Faster boot-up vì software đã có sẵn |

## 💡 Mẹo ghi nhớ cho kỳ thi AWS

- Custom AMI giúp giảm thời gian boot/configuration.
- AMI thích hợp khi cần launch nhiều EC2 instances có cùng software setup.
- Dùng AMI để prepackage software, dùng **User Data** cho customization cuối.

## ✅ Kết luận

Bài hands-on cho thấy quy trình tạo custom AMI từ một EC2 instance đã được cấu hình, rồi launch instance mới từ AMI đó. Điểm quan trọng là AMI giúp instance mới khởi động nhanh hơn vì không cần cài lại phần mềm đã được đóng gói sẵn.
