# 328. DynamoDB Session State

## 🎯 Giới thiệu
- DynamoDB không chỉ dùng để lưu dữ liệu, mà còn có thể dùng để lưu **session state** như một **cache** cho web application.
- Mục tiêu là giúp ứng dụng:
  - lưu / đọc session state theo nhu cầu,
  - chia sẻ trạng thái đăng nhập của user giữa nhiều backend web applications.

## 1. DynamoDB và ElastiCache cho session state
- Đây là hai dịch vụ có thể phục vụ cùng một mục đích: lưu **session state**.
- Điểm khác nhau chính:
  - **ElastiCache**: hoàn toàn **in-memory**
  - **DynamoDB**: **serverless**
- Cả hai đều là **key/value stores**.
- Mẹo cho exam:
  - Nếu đề bài nhấn mạnh **in-memory session state store** → nghĩ đến **ElastiCache**
  - Nếu đề bài nhấn mạnh **automatic scaling** và kiểu **serverless** → nghĩ đến **DynamoDB**

## 2. Lưu session state trên disk với EFS
- Một cách khác là lưu session state trên **disk** và chia sẻ ổ đĩa đó giữa nhiều **EC2 instances**.
- **EFS** có thể là lựa chọn tốt vì:
  - được attach vào EC2 như một **network drive**
  - có thể chia sẻ giữa nhiều EC2 instances
- Tuy nhiên:
  - **EFS** là **file system**
  - **DynamoDB** là **database**
- Vì vậy, chúng phục vụ session state theo cách khác nhau.

## 3. Các lựa chọn không phù hợp hoặc chỉ dùng local
- **EBS** và **EC2 instance store**:
  - chỉ gắn với **một EC2 instance**
  - chỉ phù hợp cho **local caching**
  - **không thể dùng cho shared caching** giữa nhiều instances
- **S3**:
  - có thể dùng cho session state, nhưng **không phải lựa chọn tốt**
  - vì **latency cao**
  - S3 phù hợp hơn cho **big files**, không phải **small objects**

## 📊 Bảng tóm tắt
| Tiêu chí | Mô tả |
|----------|------|
| DynamoDB | Dùng để lưu session state như cache; serverless; key/value store |
| ElastiCache | Dùng cho session state; hoàn toàn in-memory; key/value store |
| EFS | Lưu trên disk và chia sẻ giữa nhiều EC2 instances; là file system |
| EBS / Instance Store | Chỉ dùng cho local caching; không shared giữa nhiều instances |
| S3 | Có thể dùng nhưng latency cao; không phù hợp cho small objects |

## 💡 Mẹo ghi nhớ cho kỳ thi AWS
- **In-memory** + session state → **ElastiCache**
- **Serverless** + automatic scaling → **DynamoDB**
- **Chia sẻ disk giữa nhiều EC2** → **EFS**
- **Chỉ local trên một EC2** → **EBS / instance store**
- **S3** không phải lựa chọn tối ưu cho session state vì **latency cao**

## ✅ Kết luận
- Với session state, hai lựa chọn thường được ưu tiên là **DynamoDB** và **ElastiCache**.
- **EFS** cũng có thể dùng nếu cần chia sẻ trên disk giữa nhiều EC2 instances.
- **EBS**, **instance store**, và **S3** không phải lựa chọn tốt nhất trong bối cảnh shared session state.
