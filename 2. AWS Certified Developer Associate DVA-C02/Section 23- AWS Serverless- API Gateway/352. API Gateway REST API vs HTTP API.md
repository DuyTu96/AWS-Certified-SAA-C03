# 352. API Gateway REST API vs HTTP API

## 🎯 Giới thiệu
Trong API Gateway, ngoài **REST API** mà course đang dùng, còn có **HTTP API** và **WebSocket**.  
Ở mức độ tổng quan cho kỳ thi, bài giảng nhấn mạnh chủ yếu vào sự khác nhau giữa **REST API** và **HTTP API**.

## 1. HTTP API
- Là lựa chọn **low latency** và **cost effective** hơn.
- Dùng kiểu **AWS Lambda proxy**, **HTTP proxy API**, và **private integration**.
- Tất cả đều theo hướng **proxy**, nên **không có data mapping**.
- Chỉ hỗ trợ một số kiểu authorization:
  - **OIDC**
  - **OAuth 2.0**
- Có **built-in support for CORS**.
- **Không có**:
  - **usage plan**
  - **API keys**
- Được mô tả là:
  - **rẻ hơn**
  - **mới hơn**
  - **đơn giản hơn**
- Điểm cần nhớ cho exam:
  - **HTTP API** thường **cheaper hơn REST API**.
  - **HTTP API không hỗ trợ resource policies**.

## 2. REST API
- Đây là loại API đã được dùng xuyên suốt course.
- Có **all the features** đã học trong course.
- Khác với HTTP API ở hỗ trợ:
  - **resource policies**
- Ở mức high level, REST API là lựa chọn đầy đủ hơn về mặt tính năng so với HTTP API.

## 3. Điểm khác nhau cần nhớ cho kỳ thi
- **HTTP API**:
  - rẻ hơn
  - đơn giản hơn
  - proxy-based
  - không có data mapping
  - hỗ trợ ít authorization hơn
  - có CORS built-in
  - không có usage plan, API keys
  - không có resource policies
- **REST API**:
  - nhiều tính năng hơn
  - có resource policies
  - là loại API đã được dùng trong course

## 📊 Bảng tóm tắt
| Tiêu chí | Mô tả |
|----------|------|
| Chi phí | **HTTP API** rẻ hơn **REST API** |
| Kiến trúc | **HTTP API** thiên về **proxy** |
| Data mapping | **HTTP API** không có **data mapping** |
| Authorization | **HTTP API** chỉ hỗ trợ **OIDC** và **OAuth 2.0** |
| CORS | **HTTP API** có **built-in CORS** |
| Usage plan / API keys | **HTTP API** không có |
| Resource policies | **REST API** có, **HTTP API** không có |
| Mức độ phức tạp | **HTTP API** đơn giản hơn |
| Mức độ tính năng | **REST API** đầy đủ hơn |

## 💡 Mẹo ghi nhớ cho kỳ thi AWS
- Nhớ nhanh: **HTTP API = rẻ hơn, đơn giản hơn, proxy-only**.
- Nếu thấy nhắc đến **resource policies**, hãy nghĩ ngay đến **REST API**.
- Nếu đề bài nhấn mạnh **cost effective** và **low latency**, đó thường là dấu hiệu của **HTTP API**.
- Nếu cần **usage plan** hoặc **API keys**, transcript cho biết **HTTP API không có**.

## ✅ Kết luận
- **HTTP API** là lựa chọn **rẻ hơn và đơn giản hơn**, nhưng bị giới hạn về tính năng.
- **REST API** đầy đủ hơn và có **resource policies**.
- Với kỳ thi, điểm phân biệt quan trọng nhất là: **HTTP API cheaper**, còn **REST API hỗ trợ nhiều tính năng hơn**.
