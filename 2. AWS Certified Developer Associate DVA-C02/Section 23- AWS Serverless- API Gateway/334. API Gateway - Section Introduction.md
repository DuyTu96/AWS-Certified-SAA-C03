# 334. API Gateway - Section Introduction

## 🎯 Giới thiệu
- Tiếp nối hành trình **serverless**.
- Đã biết cách lưu dữ liệu vào **DynamoDB**.
- Câu hỏi tiếp theo là: làm sao **expose functions to the world** để người dùng truy cập ứng dụng qua **REST API** mà không cần quản lý servers.
- **API Gateway** là dịch vụ giúp đưa application ra bên ngoài dưới dạng **REST API** trên nền tảng cloud.
- Section này cũng sẽ đề cập đến **user authentication using Amazon.com**.
- Mục tiêu chung là hoàn thiện cách **deploy an API to the cloud**.

## 1. Vấn đề cần giải quyết
- Có functions và dữ liệu, nhưng cần một cách để người dùng bên ngoài truy cập.
- Không muốn tự quản lý server.
- Cần một lớp trung gian để public application thành **REST API**.

## 2. Vai trò của API Gateway
- **API Gateway** cho phép:
  - Mở rộng application ra bên ngoài.
  - Expose application như một **REST API**.
  - Hỗ trợ triển khai ứng dụng lên cloud theo hướng serverless.
- Đây là mảnh ghép giúp kết nối các phần đã học trước đó.

## 3. Nội dung sẽ học trong section này
- Cách dùng **API Gateway** để đưa application ra internet.
- Cách làm **user authentication**.
- Cách hoàn thiện quy trình **deploy an API to the cloud**.

## 💡 Mẹo ghi nhớ cho kỳ thi AWS
- Gặp bài toán **expose application as REST API** mà không muốn quản lý server, nghĩ ngay đến **API Gateway**.
- **API Gateway** thường là lớp giúp đưa các functions ra bên ngoài cho người dùng truy cập.
- Nếu đề bài nhắc đến **serverless + REST API + authentication**, đây là dấu hiệu rất mạnh của section này.
