# 339. API Gateway Stages Configurations Hands On

## 🎯 Giới thiệu
Bài này giới thiệu các **configuration options** của một **API Gateway stage**. Nội dung tập trung vào những thiết lập bạn có thể chỉnh trực tiếp trong **Stage details**, phục vụ cho việc quản trị, giám sát, và kiểm soát hành vi của stage.

## 1. Cấu hình chung của stage
- Trong **Stage details > Edit**, bạn có thể:
  - đổi **stage description**
  - bật **cache** để cache **requests** và **responses**
  - cấu hình **throttling** để giới hạn số **rate** và **burst requests per second**
- Có thể thiết lập:
  - **firewall configuration**
  - **client certificate** để xác minh request đến từ **API Gateway**

## 2. Logging, tracing và metrics
- **Logging and tracing** có thể được bật để quan sát và debug:
  - **CloudWatch Logs**
  - mức log như:
    - chỉ **errors**
    - **info logs**
    - **full request and response logs**
- Lưu ý: log chi tiết có thể hiển thị **sensitive data**
- Có thể bật:
  - **detailed metrics**
  - **custom access logging**
  - **X-Ray tracing** để tích hợp đầy đủ giữa **API Gateway** và **X-Ray service**

## 3. Các tính năng quản lý stage khác
- **Stage variables**: dùng biến theo stage như đã thấy trước đó
- **Deployment history**: xem stage đã được deploy bao nhiêu lần
- **Documentation history**: theo dõi lịch sử tài liệu của API
- **Canary**: sẽ được học kỹ hơn ở bài sau
- **Tags**: có thể gắn tag cho chính stage

## 📊 Bảng tóm tắt
| Tiêu chí | Mô tả |
|----------|------|
| Stage details | Có thể chỉnh sửa mô tả của stage |
| Cache | Cache requests và responses cho API |
| Throttling | Giới hạn rate và burst requests per second |
| Security | Có firewall configuration và client certificate |
| Logging | Dùng CloudWatch Logs với nhiều mức chi tiết |
| Tracing | Hỗ trợ X-Ray tracing |
| Metrics | Có detailed metrics và custom access logging |
| Quản lý stage | Có stage variables, deployment history, documentation history, canary, tags |

## 💡 Mẹo ghi nhớ cho kỳ thi AWS
- Nhớ nhóm theo 4 cụm chính:
  - **Performance**: cache, throttling
  - **Security**: firewall configuration, client certificate
  - **Observability**: CloudWatch Logs, detailed metrics, access logging, X-Ray
  - **Management**: stage variables, deployment history, documentation history, canary, tags
- Khi thấy câu hỏi về **debug API Gateway**, hãy nghĩ ngay đến:
  - **CloudWatch Logs**
  - **full request and response logs**
  - **X-Ray**
- Khi thấy câu hỏi về **giới hạn traffic**, liên hệ với **throttling**
- Khi cần theo dõi thay đổi stage, nhớ **deployment history** và **tags**

## ✅ Kết luận
**API Gateway stage** có nhiều tùy chọn cấu hình quan trọng để tối ưu vận hành, giám sát, và kiểm soát API. Các điểm đáng nhớ nhất là **cache**, **throttling**, **logging/tracing**, **stage variables**, và các loại **history** của stage.
