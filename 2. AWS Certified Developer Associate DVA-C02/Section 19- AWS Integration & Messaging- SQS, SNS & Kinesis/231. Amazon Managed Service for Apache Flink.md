# 231. Amazon Managed Service for Apache Flink

## 🎯 Giới thiệu
Amazon Managed Service for Apache Flink là dịch vụ dùng để xử lý **data streams theo thời gian thực**.

- Trước đây dịch vụ này có tên là **Kinesis Data Analytics for Apache Flink**
- Hiện tại đã được đổi tên thành **Amazon Managed Service for Apache Flink**
- **Flink** là một **framework**
- Thường dùng với các ngôn ngữ: **Java, SQL, Scala**

```mermaid
flowchart LR
    A[Kinesis Data Streams] --> C[Amazon Managed Service for Apache Flink]
    B[Amazon MSK<br/>(Apache Kafka)] --> C
    C --> D[Real-time stream processing]
```

## 1. Nguồn dữ liệu đầu vào
Dịch vụ này có thể đọc dữ liệu từ:

- **Kinesis Data Streams**
- **Amazon MSK**  
  - MSK là dịch vụ quản lý cho **Apache Kafka**
  - **Apache Kafka** không phải là AWS service

### ⚠️ Lưu ý quan trọng cho exam
- Flink **có thể đọc từ Kinesis Data Streams**
- Flink **không thể đọc từ Amazon Data Firehose**
- Đây có thể là **bẫy đề thi**

## 2. Những gì AWS quản lý cho bạn
Khi chạy Apache Flink application trên dịch vụ này, AWS sẽ:

- Provision **compute resources**
- Cung cấp **parallel computation**
- Hỗ trợ **automatic scaling**
- Quản lý **application backups**
  - Thực hiện qua **checkpoints**
  - Thực hiện qua **snapshots**

## 3. Mục đích sử dụng
Dịch vụ này được dùng để:

- Chạy **Apache Flink application** trên managed cluster của AWS
- Thực hiện các **transformations** trên stream data
- Tận dụng các **programming features** mà Apache Flink hỗ trợ

## 💡 Mẹo ghi nhớ cho kỳ thi AWS
- Nhớ từ khóa: **real-time stream processing**
- Nhớ nguồn vào: **Kinesis Data Streams** và **Amazon MSK**
- Nhớ bẫy: **không đọc được từ Amazon Data Firehose**
- Nhớ AWS lo phần hạ tầng: **compute, scaling, backups**
- Nếu thấy câu hỏi về xử lý stream bằng **Apache Flink**, nghĩ ngay đến **Amazon Managed Service for Apache Flink**

## ✅ Kết luận
Amazon Managed Service for Apache Flink là dịch vụ AWS dùng để chạy **Apache Flink** trên môi trường được quản lý, phục vụ **xử lý dữ liệu streaming thời gian thực**. Điểm cần nhớ nhất cho kỳ thi là nguồn dữ liệu hỗ trợ, khả năng AWS quản lý hạ tầng, và bẫy **không đọc được từ Amazon Data Firehose**.
