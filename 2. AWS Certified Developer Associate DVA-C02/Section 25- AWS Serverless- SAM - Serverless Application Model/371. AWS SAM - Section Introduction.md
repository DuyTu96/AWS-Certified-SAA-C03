# 371. AWS SAM - Section Introduction

## 🎯 Giới thiệu
- Trong bài này, mục tiêu là hiểu cách triển khai một API theo kiểu **serverless** lên cloud.
- Nếu chỉ dùng **CloudFormation** để quản lý toàn bộ thứ này thì sẽ khá phức tạp.
- AWS giới thiệu **SAM** để đơn giản hóa việc mô tả cách ứng dụng **behave** và **deploy** bằng file **YAML**.

## 1. AWS SAM là gì?
- **SAM** là một cách để viết **YAML files** và **templates** mô tả ứng dụng sẽ hoạt động và được triển khai như thế nào.
- Có thể xem SAM như một **shortcut** để làm việc với **CloudFormation**.

## 2. Vì sao dùng SAM?
- SAM giúp việc xử lý trở nên **natural** hơn với developer.
- Nó làm cho quá trình mô tả và triển khai application trở nên dễ tiếp cận hơn so với cách làm trực tiếp bằng CloudFormation.
- AWS đang xem SAM là một service ngày càng phổ biến.

## 3. Ý nghĩa với kỳ thi AWS
- Trong exam, SAM chỉ xuất hiện ở mức câu hỏi **cơ bản**.
- Bạn chỉ cần nắm ý chính:
  - SAM dùng để làm gì
  - SAM liên hệ thế nào với **CloudFormation**
  - SAM giúp đơn giản hóa việc deploy serverless applications

## 💡 Mẹo ghi nhớ cho kỳ thi AWS
- Nhớ nhanh: **SAM = Serverless Application Model** theo tinh thần của bài giảng này, và nó là cách viết **YAML** để hỗ trợ deploy dễ hơn.
- Nếu thấy câu hỏi về:
  - **serverless deployment**
  - **CloudFormation quá phức tạp**
  - **developer-friendly templates**
  
  thì nghĩ ngay đến **SAM**.
- Với exam, không cần đi quá sâu ở phần này, chỉ cần hiểu vai trò và mối quan hệ với **CloudFormation**.

## ✅ Kết luận
- **AWS SAM** được giới thiệu như một cách đơn giản hơn để mô tả và triển khai ứng dụng serverless.
- Nó giúp developer làm việc tự nhiên hơn so với chỉ dùng **CloudFormation**.
- Phần này trong exam chủ yếu ở mức nhận biết cơ bản, nhưng sẽ hữu ích cho các bài học thực chiến về sau.
