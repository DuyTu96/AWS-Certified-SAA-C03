# 384. Cognito Overview

## 🎯 Giới thiệu
Amazon Cognito được dùng để cấp **identity** cho người dùng tương tác với **web** và **mobile application**.

- Các user này thường nằm **ngoài AWS account**, nên Cognito phù hợp cho **application users** chứ không phải IAM users.
- Khi ôn thi, cần nhớ các keyword như:
  - **hundreds of users**
  - **mobile users**
  - **authentication**
  - cơ chế như **SAML**

## 1. Cognito User Pool
**Cognito User Pool** cung cấp chức năng **sign-in** cho app users.

- Dùng cho việc đăng nhập người dùng của ứng dụng
- Có tích hợp tốt với:
  - **API Gateway**
  - **Application Load Balancer (ALB)**

## 2. Cognito Identity Pool
**Cognito Identity Pool** trước đây được gọi là **Federated Identity**.

- Dùng để cấp **temporary AWS credentials**
- Áp dụng cho user đã đăng ký trong ứng dụng
- Mục đích là để user có thể truy cập **AWS resources trực tiếp**
- Có tích hợp tốt với **Cognito User Pools**

## 3. Cognito vs IAM
Nếu tự hỏi tại sao không dùng luôn user trong **IAM**:

- **IAM users** là user trong AWS
- **Cognito** dành cho **web/mobile application users** ở bên ngoài AWS
- Vì vậy Cognito giải quyết bài toán identity cho người dùng ứng dụng, không phải quản trị identity nội bộ AWS

## 📊 Bảng tóm tắt
| Tiêu chí | Mô tả |
|----------|------|
| Mục tiêu của Cognito | Cấp **identity** cho user của web/mobile app |
| Đối tượng sử dụng | User nằm **ngoài AWS account** |
| **Cognito User Pool** | Cung cấp **sign-in** cho app users |
| Tích hợp của User Pool | **API Gateway**, **Application Load Balancer** |
| **Cognito Identity Pool** | Cấp **temporary AWS credentials** |
| Tên cũ của Identity Pool | **Federated Identity** |
| Mục đích của Identity Pool | Cho phép user truy cập AWS resources trực tiếp |
| So với IAM | IAM cho user trong AWS, Cognito cho user ứng dụng |

## 💡 Mẹo ghi nhớ cho kỳ thi AWS
- **User Pool = sign-in / authentication cho app users**
- **Identity Pool = temporary AWS credentials**
- Nhớ câu hỏi thi thường xoay quanh:
  - user là **mobile users**
  - hệ thống có **hundreds of users**
  - cần **authentication** với **SAML** hoặc cơ chế tương tự
- Nếu đề bài nói về user của ứng dụng web/mobile, hãy nghĩ ngay đến **Cognito** thay vì **IAM**

## ✅ Kết luận
Amazon Cognito là dịch vụ dành cho **web và mobile application users** ở bên ngoài AWS.  
- **User Pool** dùng cho **sign-in**
- **Identity Pool** dùng để cấp **temporary AWS credentials**
- Cognito có tích hợp tốt với **API Gateway**, **ALB**, và **Cognito User Pools**
