# 364. CodeBuild Hands On Part I

## 🎯 Giới thiệu
- Bài học này giới thiệu cách dùng **CodeBuild** để kiểm tra ứng dụng trước khi deploy, thay vì chỉ dùng để build.
- Mục tiêu thực hành là xác nhận thông điệp **"Congratulations"** vẫn xuất hiện trong project.
- Tác giả tạo một **CodeBuild project** mới và cấu hình để lấy source từ **GitHub**.

## 1. Tạo CodeBuild Project
- Tạo project mới trong CodeBuild với tên **`MyFirstBuild`**.
- Không cần:
  - Description
  - Build Badge
  - Tags
- Chọn **Source = GitHub** và kết nối bằng **OAuth**.
- Sau khi authorize, CodeBuild được connect với GitHub và có thể chọn repository **nodejs app**.

## 2. Cấu hình Build
- Build từ **main branch**.
- Không cần Pull Request hay Release ở bước này.
- Thiết lập để build lại mỗi khi có **code change pushed** lên repository.
- Filter Group được dùng là **PUSH**.
- Chọn **Single Build**.

## 3. Môi trường, role và buildspec
- Chọn **Managed Image**.
- Chọn OS **Ubuntu**.
- Runtime: **Standard**.
- Image: dùng bản latest, ví dụ **standard 7.0**.
- Tạo **Service Role** mới với tên **`CodeBuildDemoServiceRole`**.
- Các điểm cấu hình quan trọng:
  - **Timeout** mặc định là 1 giờ, có thể chỉnh xuống ví dụ 10 phút.
  - Có thể chọn **VPC** để CodeBuild truy cập database hoặc tài nguyên trong VPC.
  - Compute Size đang là **3 GB memory và 2 vCPU**.
  - Có thể khai báo **Environment Variables**.
- Dùng **Buildspec.yaml** để chứa lệnh build.
- File **Buildspec.yaml** phải nằm trong **Source Code Root Directory**.
- Artifacts không cần thiết trong bài này vì không build ra sản phẩm.
- Có thể xuất log ra **CloudWatch logs** hoặc **Amazon S3**.

## 4. Kết quả build
- Sau khi tạo project, bắt đầu **Start build**.
- CodeBuild sẽ pull repository từ GitHub.
- Build thất bại vì **không có file yaml** phù hợp để lấy **Buildspec**.
- Lý do là chưa upload **Buildspec.yaml** vào thư mục source.
- Đây là trạng thái mong đợi ở thời điểm này, vì file sẽ được thêm ở bài sau.
- Kết luận: project CodeBuild đầu tiên đã được thiết lập thành công và sẵn sàng cho bước tiếp theo.

## 💡 Mẹo ghi nhớ cho kỳ thi AWS
- **CodeBuild** có thể chạy build/test trực tiếp từ source như **GitHub**.
- Muốn build được, thường cần **Buildspec.yaml**.
- Có thể cấu hình:
  - **Managed Image**
  - **Timeout**
  - **VPC**
  - **Environment Variables**
  - **Artifacts**
- Khi thấy lỗi không tìm được buildspec, hãy nghĩ ngay đến việc **file Buildspec.yaml chưa có trong source**.
- Trường hợp này nhấn mạnh rằng CodeBuild có thể được dùng cho cả **test automation** chứ không chỉ build artifact.

- Đây là nền tảng để sang bài tiếp theo và hoàn thiện luồng thực hành CodeBuild.
