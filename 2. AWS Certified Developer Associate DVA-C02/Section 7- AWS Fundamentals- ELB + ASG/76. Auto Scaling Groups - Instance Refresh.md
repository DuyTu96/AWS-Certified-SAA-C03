# 76. Auto Scaling Groups - Instance Refresh

## 🎯 Giới thiệu

Bài học giới thiệu **Instance Refresh** — một tính năng hữu ích của **Auto Scaling Groups** để cập nhật toàn bộ ASG bằng launch template mới.

Use case chính:

- Bạn tạo launch template mới.
- Muốn recreate toàn bộ EC2 instances trong ASG.
- Ví dụ: cập nhật underlying AMI của EC2 instances.

## 1. 🔄 Instance Refresh là gì?

**Instance Refresh** cho phép cập nhật toàn bộ Auto Scaling Group mà không cần tự terminate từng instance thủ công.

Thay vì:

- Terminate instance.
- Chờ instance mới được tạo lại.

Ta dùng native feature:

- **Instance Refresh**.

## 2. 🧱 Old Launch Template và New Launch Template

Ví dụ trong bài:

- ASG hiện tại có EC2 instances được launch bằng old launch template.
- Bạn tạo new launch template.
- New launch template có thể chứa AMI mới.

Sau đó gọi API:

- `Start Instance Refresh`

```mermaid
flowchart LR
    Old[Old Launch Template Instances] --> IR[Start Instance Refresh]
    IR --> Term[Terminate Old Instances]
    Term --> New[Launch New Instances]
    New --> LT[New Launch Template]
```

## 3. 🩺 Minimum Healthy Percentage

Khi chạy Instance Refresh, có thể đặt:

- **Minimum healthy percentage**.

Ví dụ trong bài:

- 60%.

Ý nghĩa:

- Cho ASG biết bao nhiêu instances cần giữ healthy trong quá trình refresh.
- Gián tiếp quyết định có thể delete bao nhiêu instances over time.

## 4. 🚀 Quá trình Refresh

Trong quá trình Instance Refresh:

1. Một số old instances bị terminated.
2. New instances được launch bằng new launch template.
3. Theo thời gian, toàn bộ instances dùng old launch template bị thay thế.
4. New instances dùng launch template mới.

## 5. ⏱️ Warm-up Time

Có thể specify **warm-up time**.

Ý nghĩa:

- Khoảng thời gian chờ để EC2 instance mới sẵn sàng.
- Sau warm-up time, ASG có thể assume instance mới ready to use.

## 📊 Bảng tóm tắt

| Tiêu chí | Mô tả |
|----------|------|
| Tính năng | Instance Refresh |
| Dịch vụ | Auto Scaling Groups |
| Mục tiêu | Recreate toàn bộ EC2 instances bằng launch template mới |
| API | Start Instance Refresh |
| Use case | Update AMI / launch template |
| Minimum healthy percentage | Quy định tỷ lệ healthy cần giữ trong quá trình refresh |
| Warm-up time | Thời gian chờ instance mới sẵn sàng |

## 💡 Mẹo ghi nhớ cho kỳ thi AWS

- Muốn update toàn bộ ASG theo launch template mới → dùng **Instance Refresh**.
- **Minimum healthy percentage** kiểm soát mức độ an toàn khi thay thế instances.
- **Warm-up time** giúp instance mới có thời gian sẵn sàng trước khi tiếp tục refresh.

## ✅ Kết luận

**Instance Refresh** giúp cập nhật toàn bộ EC2 instances trong Auto Scaling Group bằng launch template mới, thay thế dần old instances trong khi vẫn duy trì tỷ lệ healthy tối thiểu và cho instances mới thời gian warm up.
