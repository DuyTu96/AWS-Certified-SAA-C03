# 394. Step Functions - Invoke Lambda - Hands On

## 🎯 Giới thiệu
Bài thực hành này ցույց cách tạo một **Step Functions state machine** để **invoke Lambda**, sau đó dùng **Choice state** để rẽ nhánh theo kết quả trả về từ Lambda.

Mục tiêu chính:
- Tạo state machine từ **blank format**
- Dùng **Lambda Invoke** để gọi Lambda function
- Dùng **Choice state** để kiểm tra output
- Quan sát từng bước execution trong Step Functions

```mermaid
flowchart TD
    A[Start execution] --> B[Lambda Invoke]
    B --> C[Choice State]
    C -->|Output contains "Stephane"| D[Is Teacher]
    C -->|Output does not contain "Stephane"| E[Not Teacher]
    D --> F[Output: Woohoo!]
    E --> G[Error: Stephane the teacher wasn't found]
```

## 1. Tạo state machine và flow điều khiển
- Tạo **state machine** mới bằng **Create state machine**
- Chọn **blank format** để tự xây workflow
- Có thể dựng flow bằng **designer** hoặc vào **Code** để dán JSON có sẵn
- Flow trong bài gồm:
  - **Lambda Invoke**
  - **Choice state**
  - Hai nhánh:
    - **Is Teacher**
    - **Not Teacher**
- Logic kiểm tra:
  - Nếu output của Lambda chứa **Stephane** thì đi vào **Is Teacher**
  - Nếu không có thì đi vào **Not Teacher**

## 2. Tạo và test Lambda function
- Tạo Lambda function từ đầu với tên **HelloFunction**
- Chọn **latest Node.js runtime**
- Code Lambda rất đơn giản:
  - Lấy key `who` từ event
  - Trả về lời chào theo giá trị của `who`
- Test Lambda với payload:
  - `who = John` → trả về **Hello, John!**
  - `who = Stephane` → trả về **Hello, Stephane!**
  - `who = Alice` → trả về **Hello, Alice!**
- Sau khi test thành công:
  - Copy **ARN** của Lambda
  - Dán ARN vào **Lambda Invoke stage** tại phần **Enter function name**

## 3. Chạy state machine và quan sát kết quả
- Tạo state machine xong thì start execution
- Step Functions tự tạo **execution role** để cấp quyền cần thiết cho state machine
- Chạy với payload:
  - `who = Stephane Maarek`
  - Kết quả:
    - Lambda Invoke nhận input này
    - Lambda trả về **Hello, Stephane Maarek!**
    - Choice state kiểm tra thấy **Stephane**
    - Đi vào **Is Teacher**
    - Output là **Woohoo!**
- Chạy lại với:
  - `who = John Doe`
  - Kết quả:
    - Lambda trả về **Hello, John Doe!**
    - Choice state không tìm thấy **Stephane**
    - Đi vào **Not Teacher**
    - Trả về error code: **Stephane the teacher wasn't found**
- Có thể xem từng event trong execution để hiểu:
  - State nào đã chạy
  - Mất bao lâu
  - Flow đã rẽ nhánh như thế nào

## 💡 Mẹo ghi nhớ cho kỳ thi AWS
- **Step Functions** có thể điều phối nhiều step, trong bài là gọi **Lambda** rồi rẽ nhánh bằng **Choice state**
- **Lambda Invoke** dùng ARN của function để state machine gọi đúng Lambda
- **Choice state** dựa trên output của bước trước để quyết định luồng tiếp theo
- Nếu state machine cần quyền để gọi Lambda, **execution role** sẽ được tạo để cấp permission cần thiết
- Khi ôn thi, nhớ luồng tư duy:
  - **Input vào Lambda**
  - **Lambda trả output**
  - **Choice state kiểm tra output**
  - **Đi tiếp nhánh đúng**

## ✅ Kết luận
Bài hands-on này minh họa cách xây một **Step Functions state machine** rất cơ bản nhưng quan trọng: gọi **Lambda**, xử lý output, rồi dùng **Choice state** để quyết định nhánh tiếp theo. Đây là mẫu workflow rất đáng nhớ khi ôn thi AWS vì nó thể hiện rõ cách Step Functions orchestration một request flow từ đầu đến cuối.
