# 203. CloudFormation - Intrinsic Functions

## 🎯 Giới thiệu
- **Intrinsic Functions** là các hàm tích hợp sẵn trong **CloudFormation**.
- Bài này nhấn mạnh các hàm quan trọng nhất cho kỳ thi, đặc biệt là:
  - `Ref`
  - `Fn::GetAtt`
  - `FindInMap`
  - `ImportValue`
  - `Base64`
  - `Condition Functions` như `If`, `Not`, `Equals`, `And`, `Or`
- Các hàm này giúp template linh hoạt hơn khi tham chiếu tài nguyên, lấy thuộc tính, nhập giá trị từ stack khác, hoặc tạo điều kiện triển khai.

## 1. `Ref` và `Fn::GetAtt`
- `Ref` dùng để lấy **reference**:
  - Với **parameter**: trả về giá trị của parameter.
  - Với **resource**: trả về **physical ID** của resource đã được tạo, ví dụ `EC2 instance`.
- Cú pháp rút gọn của `Ref` là `!Ref`.
- `Fn::GetAtt` dùng để lấy **attribute** của resource.
- Chỉ lấy được những attribute mà AWS **documented** cho resource đó.
- Ví dụ với `EC2 instance`, có thể lấy:
  - `PrivateDnsName`
  - `PrivateIp`
  - `PublicDnsName`
  - `PublicIp`
  - `VpcIp`
- Ứng dụng:
  - Tạo `Route53 CNAME record` bằng cách lấy `PublicDnsName` của `EC2 instance`.

## 2. `FindInMap` và `ImportValue`
- `FindInMap`:
  - Dùng để lấy giá trị từ một `map` dựa trên một `key` cụ thể.
  - Chỉ xuất hiện trong ngữ cảnh **mappings**.
- `ImportValue`:
  - Dùng để import giá trị đã được **export** từ stack khác.
  - Ví dụ: import `Security Group ID` tên `SSHSecurityGroup` từ một stack khác để gắn vào `EC2 instance`.
- Ý nghĩa trong CloudFormation:
  - Giúp chia nhỏ hạ tầng thành nhiều stack.
  - Tái sử dụng tài nguyên và giá trị giữa các stack.

## 3. `Base64` và `Condition Functions`
- `Base64`:
  - Dùng để chuyển một string sang dạng **Base64**.
  - Thường dùng khi truyền dữ liệu vào `UserData` của `EC2 instance`.
  - Chuỗi `UserData` sẽ được mã hóa rồi gửi cho instance.
- `Condition Functions`:
  - Dùng để tạo điều kiện trong template và quyết định có tạo resource hay không.
  - Các hàm được nhắc đến:
    - `If`
    - `Not`
    - `Equals`
    - `And`
    - `Or`

## 📊 Bảng tóm tắt
| Tiêu chí | Mô tả |
|----------|------|
| `Ref` | Lấy reference của parameter hoặc physical ID của resource |
| `Fn::GetAtt` | Lấy attribute đã được AWS document của resource |
| `FindInMap` | Lấy value từ `mapping` theo key |
| `ImportValue` | Import giá trị được export từ stack khác |
| `Base64` | Mã hóa string sang Base64, thường dùng cho `UserData` |
| `Condition Functions` | Dùng để tạo điều kiện triển khai resource |

## 💡 Mẹo ghi nhớ cho kỳ thi AWS
- `Ref` = “tham chiếu trực tiếp”
  - Parameter thì ra value
  - Resource thì ra physical ID
- `GetAtt` = “lấy thuộc tính”
  - Chỉ lấy được thuộc tính AWS cho phép
- `ImportValue` thường đi với stack khác và `Export`
- `Base64` hay gặp nhất trong `UserData` của `EC2`
- `Condition Functions` dùng khi cần tạo resource có điều kiện
- Khi làm bài thi, hãy nhớ:
  - `Ref` và `GetAtt` là hai hàm cực hay xuất hiện nhất
  - Muốn lấy `PublicDnsName` hay `PrivateIp` của `EC2` thì nghĩ đến `Fn::GetAtt`

## ✅ Kết luận
- `Intrinsic Functions` là nền tảng quan trọng của **CloudFormation**.
- Trong bài này, trọng tâm là:
  - `Ref` để lấy reference hoặc physical ID
  - `Fn::GetAtt` để lấy attribute của resource
  - `FindInMap` để tra giá trị trong mapping
  - `ImportValue` để dùng dữ liệu từ stack khác
  - `Base64` để mã hóa dữ liệu cho `UserData`
  - `Condition Functions` để tạo logic triển khai có điều kiện
