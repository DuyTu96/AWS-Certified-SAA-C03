# 118. S3 Website Hands On

## 🎯 Giới thiệu

Bài thực hành bật static website hosting cho S3 bucket, upload `index.html`, và truy cập website qua bucket website endpoint.

## 1. 📂 Chuẩn bị files trong Bucket

Trong bucket đã có `coffee.jpg`.

Các bước bổ sung:

- Upload thêm `beach.jpg`.
- Bucket lúc này có hai files.

## 2. 🌐 Enable Static Website Hosting

Vào tab Properties:

- Scroll xuống Static website hosting.
- Chọn Edit.
- Enable static website hosting.
- Chọn host a static website.
- Nhập index document: `index.html`.

⚠️ Console hiển thị warning:

- Muốn dùng website endpoint thì toàn bộ content cần publicly readable.
- Điều này đã được cấu hình trong bài trước bằng S3 Bucket Policy.

## 3. 🚀 Upload Index Document và Test Website

Sau khi bật static website hosting:

- Upload file `index.html`.
- Quay lại Properties.
- Copy bucket website endpoint.
- Mở URL.

Kết quả:

- Website hiển thị nội dung `I love coffee. Hello world!`.
- Image `coffee.jpg` hiển thị được.
- Mở image trong tab mới sẽ thấy public URL của `coffee.jpg`.
- Có thể đổi sang `beach.jpg` và xem được image beach.

| Files trong demo | `coffee.jpg`, `beach.jpg`, `index.html` |
| Setting | Static website hosting |
| Index document | `index.html` |

| Điều kiện | Content publicly readable |
| Kết quả | Website và images hiển thị qua internet |

## 💡 Mẹo ghi nhớ cho kỳ thi AWS

- Static website hosting cần index document.
- Website endpoint chỉ hoạt động đúng khi files có thể public read.
- S3 Bucket Policy public từ bài trước giúp website đọc được images và HTML.

## ✅ Kết luận

Bài thực hành cho thấy cách biến một S3 bucket thành static website: bật hosting, cấu hình `index.html`, upload file và truy cập qua bucket website endpoint.
