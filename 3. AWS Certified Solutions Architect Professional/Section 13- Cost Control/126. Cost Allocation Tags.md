# 126. Cost Allocation Tags

## 🎯 Giới thiệu
AWS Cost Allocation Tags là cơ chế dùng tags để phục vụ **cost allocation** và tạo **detailed costing reports**. Khi xem **billing reports**, các tag này sẽ xuất hiện như **additional columns**, giúp phân chia dữ liệu chi phí chi tiết hơn. ⏱️

## 1. Mục đích của Cost Allocation Tags
- Dùng để **theo dõi và phân loại chi phí** giữa các tài nguyên.
- Giúp **group data** theo tag khi xem báo cáo billing.
- Hữu ích khi xây dựng **tagging strategy** trong AWS accounts, đặc biệt là **cross accounts**.

## 2. Hai loại Cost Allocation Tags
- **AWS Generated Cost Allocation Tags**
  - AWS tự động áp dụng cho resource bạn tạo.
  - Bắt đầu bằng prefix **`AWS`**.
  - Ví dụ: **`AWS created by`**.
  - Không áp dụng cho các resource đã được tạo **trước khi bật** tính năng này.

- **User tags**
  - Do **bạn tự tạo**.
  - Bắt đầu bằng prefix **`user`**.

## 3. Cách hoạt động trong Billing
- Cost allocation tags chỉ xuất hiện trong **billing console**.
- Không hiển thị ở nơi khác.
- Cần khoảng **24 hours** để tag xuất hiện trong reports sau khi được kích hoạt.

## 📊 Bảng tóm tắt
| Tiêu chí | Mô tả |
|----------|------|
| Mục đích | Phân tích và chia nhỏ chi phí theo tag |
| Kết quả trên billing report | Hiển thị như các cột bổ sung |
| Loại 1 | **AWS Generated Cost Allocation Tags** - prefix `AWS` |
| Loại 2 | **User tags** - prefix `user` |
| Phạm vi hiển thị | Chỉ trong **billing console** |
| Thời gian cập nhật | Khoảng **24 hours** |
| Lưu ý quan trọng | Cần có **tagging strategy** cho AWS accounts và cross accounts |

## 💡 Mẹo ghi nhớ cho kỳ thi AWS
- Nhớ rằng **Cost Allocation Tags = phục vụ billing/cost report**, không phải để quản lý resource thông thường.
- **AWS-generated** có prefix **`AWS`**, còn **user-created** có prefix **`user`**.
- Tag chỉ xuất hiện trong **billing console** và cần **24 giờ** để phản ánh.
- Không quên: tag này **không áp dụng ngược** cho resource đã tồn tại trước khi bật tính năng.

## ✅ Kết luận
Cost Allocation Tags là công cụ quan trọng để **theo dõi chi phí AWS theo tag**. Điểm cần nhớ cho kỳ thi là: có **2 loại tag**, xuất hiện trong **billing console**, mất khoảng **24 giờ** để thấy trong report, và cần có **tagging strategy** rõ ràng để phân tích chi phí hiệu quả.
