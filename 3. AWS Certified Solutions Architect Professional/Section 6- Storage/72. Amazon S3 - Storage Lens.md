# 72. Amazon S3 - Storage Lens

## 🎯 Giới thiệu
- **Amazon S3 Storage Lens** giúp bạn **understand, analyze, and optimize** storage trên toàn bộ **AWS Organization**.
- Mục tiêu chính:
  - Phát hiện **anomalies**
  - Tìm **cost efficiencies**
  - Áp dụng **data protection best practices**
- Dữ liệu có thể được:
  - Tổng hợp ở mức **organization**
  - Hoặc theo **accounts, regions, buckets, prefixes**
- Có thể dùng:
  - **Default dashboard** do Storage Lens cung cấp
  - Hoặc tạo **custom dashboard**
- Các metrics và reports có thể export ra **S3 bucket** dưới dạng **CSV** hoặc **Parquet**

## 1. Dashboard và phạm vi dữ liệu 📊
- Storage Lens có **default dashboard** với:
  - **summarized insights**
  - **trends** cho cả **free** và **advanced metrics**
- Dashboard mặc định hiển thị dữ liệu trên:
  - **multiple regions**
  - **multiple accounts**
- Đây là cấu hình **centrally managed**
- Bạn có thể chọn:
  - region
  - accounts
  - buckets
  - storage classes
- **Không thể delete default dashboard**, nhưng có thể **disable** nó

## 2. Các nhóm metrics chính 🔎
- **Summary metrics**
  - Cung cấp insight tổng quan về S3 storage
  - Ví dụ:
    - **storage bytes**
    - **object counts**
- Use case:
  - tìm bucket/prefix **tăng nhanh**
  - hoặc **không được sử dụng**
- **Cost optimization metrics**
  - Tập trung vào tối ưu chi phí storage
  - Ví dụ:
    - **non-current version storage bytes**
    - **incomplete multi-part upload storage bytes**
- Use case:
  - phát hiện **failed multi-part uploads**
  - tìm objects có thể chuyển sang **lower-cost storage classes**
- **Data protection metrics**
  - Theo dõi các tính năng bảo vệ dữ liệu
  - Ví dụ:
    - **versioning enabled bucket counts**
    - **MFA delete enabled bucket counts**
    - **SSE-KMS enabled bucket counts**
    - **cross replication region rules counts**
- Use case:
  - phát hiện bucket **không tuân thủ best practices**
- **Access management metrics**
  - Cung cấp insight về **S3 bucket ownership**
- **Event metrics**
  - Theo dõi **S3 event notifications**
- **Performance metrics**
  - Theo dõi **S3 Transfer Acceleration**
- **Activity metrics**
  - Bao gồm:
    - all requests
    - get requests
    - put requests
    - bytes downloaded
- **HTTP status code metrics**
  - Ví dụ:
    - **200 OK**
    - **403 Forbidden**
  - Giúp hiểu kiểu usage của buckets

## 3. Free vs Advanced Metrics 💰
- **Free metrics**
  - Tự động có sẵn cho mọi customer
  - Khoảng **28 usage metrics**
  - Dữ liệu query được trong **14 days**
- **Advanced metrics and recommendations**
  - Có thêm:
    - **activity**
    - **advanced cost optimization**
    - **advanced data protection**
    - **status codes**
  - Metrics được publish sang **CloudWatch**
  - Có thể truy cập mà **không phát sinh thêm charges**
  - Có thể collect metrics đến **prefix level**
  - Dữ liệu có sẵn trong **15 months**

## 💡 Mẹo ghi nhớ cho kỳ thi AWS
- Nhớ cụm: **Organization-wide visibility**
- Nhớ 3 ý lớn:
  - **Analyze**
  - **Optimize**
  - **Protect**
- Phân biệt:
  - **Free metrics**: cơ bản, 14 days
  - **Advanced metrics**: nhiều hơn, 15 months, có **CloudWatch**
- Default dashboard:
  - có sẵn
  - multi-account
  - multi-region
  - **cannot delete, can disable**
- Câu hỏi thi thường xoay quanh:
  - **prefix-level metrics**
  - **CSV/Parquet export**
  - **cost optimization**
  - **data protection best practices**

## ✅ Kết luận
- **S3 Storage Lens** là service để theo dõi, phân tích và tối ưu S3 storage ở quy mô lớn.
- Nó cung cấp dashboard, metrics theo nhiều mức, và khả năng export báo cáo để hỗ trợ:
  - **analysis**
  - **cost control**
  - **data protection**
- Khi ôn thi, hãy nhớ rõ sự khác nhau giữa **free** và **advanced metrics**, cùng với khả năng quan sát trên **multiple accounts** và **multiple regions**.
