# 84. Lambda@Edge Reduce Latency

## 🎯 Giới thiệu
- `Lambda@Edge` được dùng để **redirect request sang origin phù hợp hơn** nhằm **giảm latency** cho các request đi qua `CloudFront Edge location`.
- Mục tiêu chính trong transcript: giảm độ trễ của **first call** khi object chưa có trong cache của `CloudFront`.

## 1. Vấn đề latency với kiến trúc S3 + CloudFront
- Kiến trúc ban đầu:
  - `S3 bucket` làm origin
  - `CloudFront Edge location` ở phía trước
- Nếu Edge location ở xa `S3 bucket` thì:
  - request đầu tiên để lấy file có thể mất nhiều thời gian
  - khi object đã cache sẵn thì response rất nhanh
- Điểm nghẽn nằm ở việc **CloudFront phải đi request về origin xa**, làm tăng latency cho lần truy cập đầu.

```mermaid
flowchart LR
    U[Users] --> CF[CloudFront Edge Location]
    CF --> S3A[S3 Bucket 1\n(us-east-1)]
    CF --> CACHE[Cache hit\nResponse nhanh]
    S3A --> CF
```

## 2. Cách giảm latency bằng Lambda@Edge và origin khác nhau
- Giải pháp trong transcript:
  - Tạo **S3 bucket thứ hai** ở `EU West 2`
  - Thiết lập **S3 cross region replication** giữa 2 bucket
  - Dùng `Lambda@Edge` để kiểm tra request đến từ đâu
- Logic của `Lambda@Edge`:
  - Nếu request đến từ region gần bucket 1 thì route về bucket 1
  - Nếu request đến từ region gần bucket 2 thì **modify origin request** và chuyển sang origin thứ hai
- Vì content được replicate giống nhau ở cả hai region nên có thể chọn origin gần user hơn để giảm latency.

```mermaid
flowchart TD
    U[User request] --> CF[CloudFront Edge Location]
    CF --> L[Lambada@Edge\ncheck request source]
    L -->|Closer to bucket 1| O1[S3 Bucket 1]
    L -->|Closer to bucket 2| O2[S3 Bucket 2\nEU West 2]
    O1 --> CF
    O2 --> CF
    CF --> R[Object returned\nLatency reduced]
```

## 3. Ý nghĩa cho ôn thi AWS
- Đây là một **pattern tối ưu latency** cho request đầu tiên vào `CloudFront Edge location`.
- `Lambda@Edge` được dùng để:
  - nhận biết nguồn request
  - thay đổi origin request
  - route đến origin gần hơn
- `S3 cross region replication` là yếu tố giúp dữ liệu **giống nhau ở nhiều region**, để có thể phục vụ từ origin gần user hơn.
- Kết quả:
  - giảm độ trễ của **first call**
  - cải thiện trải nghiệm khi object chưa có trong cache

## 💡 Mẹo ghi nhớ cho kỳ thi AWS
- Nhớ công thức: **CloudFront + Lambda@Edge + multiple origins + replication = giảm latency**.
- Nếu đề bài nói về:
  - **first request chậm**
  - **origin xa**
  - **route sang bucket gần user hơn**
  - **modify origin request**
  
  thì nghĩ ngay đến `Lambda@Edge`.
- `Cache hit` thì nhanh; vấn đề chỉ xuất hiện mạnh ở **cache miss / first call**.

## ✅ Kết luận
- `Lambda@Edge` có thể giúp giảm latency bằng cách **điều hướng request đến origin gần hơn**.
- Khi kết hợp với `S3 cross region replication`, cùng một content có thể được phục vụ từ nhiều region.
- Đây là một pattern hữu ích để tối ưu trải nghiệm truy cập `CloudFront Edge location`, đặc biệt cho request đầu tiên.
