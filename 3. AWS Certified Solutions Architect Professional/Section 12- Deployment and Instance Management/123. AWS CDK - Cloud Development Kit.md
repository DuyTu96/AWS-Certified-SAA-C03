# 123. AWS CDK - Cloud Development Kit

## 🎯 Giới thiệu
AWS CDK (Cloud Development Kit) là cách để định nghĩa cloud infrastructure bằng một ngôn ngữ lập trình quen thuộc như **JavaScript/TypeScript, Python, Java, .NET**.

- Không cần viết trực tiếp **CloudFormation YAML** thủ công.
- CDK sẽ **compile** code của bạn thành **CloudFormation template** ở dạng **JSON hoặc YAML**.
- Nhờ đó, có thể triển khai **infrastructure** và **application runtime code** bằng cùng một ngôn ngữ.
- Đặc biệt hữu ích cho:
  - **Lambda**
  - **Docker containers** trên **ECS** và **EKS**

## 1. Cách CDK hoạt động
CDK cho phép bạn viết một **CDK application** bằng ngôn ngữ lập trình quen thuộc, sau đó dùng **CDK CLI** để chuyển code đó thành **CloudFormation template**.

```mermaid
flowchart LR
    A[CDK Application\n(JavaScript / TypeScript / Python / Java / .NET)] --> B[CDK CLI]
    B --> C[Generated CloudFormation Template\n(JSON / YAML)]
    C --> D[CloudFormation]
    D --> E[Deploy Infrastructure]
```

- Bạn định nghĩa các tài nguyên như:
  - **Lambda function**
  - **DynamoDB table**
  - **VPC**
  - **ECS cluster**
  - **Application Load Balancer**
  - **Fargate service**
- Sau đó CDK tạo ra **CloudFormation template** tương ứng.
- Template này được đưa vào **CloudFormation** để deploy infrastructure.

## 2. Vì sao dùng CDK
CDK được giới thiệu như một lựa chọn thay thế thuận tiện hơn so với việc viết CloudFormation trực tiếp.

Các lợi ích được nhắc đến trong transcript:
- Dùng ngôn ngữ lập trình quen thuộc thay vì YAML
- Có **type safety**
- Dễ dùng các **constructs** quen thuộc
- Viết nhanh hơn
- Có thể **reuse code**
- Hỗ trợ các cấu trúc như **for loops**

## 3. Ví dụ trong bài giảng
Transcript đưa ví dụ:
- Chọn **Python** để viết CDK application
- Định nghĩa:
  - một **Lambda function**
  - một **DynamoDB table**
  - các dịch vụ AWS khác
- Dùng **CDK CLI** để transform thành **CloudFormation template**
- Sau đó deploy qua **CloudFormation**

Một ví dụ khác trong transcript dùng **JavaScript/TypeScript**:
- **VPC**
- **ECS cluster**
- **Application Load Balancer**
- **Fargate service**

## 💡 Mẹo ghi nhớ cho kỳ thi AWS
- CDK = viết **infrastructure as code** bằng **programming language** quen thuộc.
- CDK **không thay thế CloudFormation** hoàn toàn, mà **generate CloudFormation template**.
- Câu hỏi thi thường xoay quanh:
  - CDK dùng để làm gì?
  - CDK output ra gì?
  - CDK phù hợp với use case nào?
- Nhớ cụm: **Code bằng language quen thuộc -> CDK CLI -> CloudFormation template -> CloudFormation deploy**.

## ✅ Kết luận
AWS CDK giúp bạn định nghĩa và triển khai cloud infrastructure bằng ngôn ngữ lập trình thay vì viết CloudFormation trực tiếp. Điểm cốt lõi cần nhớ là CDK **compile thành CloudFormation template**, rất phù hợp khi muốn kết hợp **infrastructure** và **runtime code** trong cùng một hệ sinh thái phát triển.
