# 305. Web Application Firewall (WAF)

## 🎯 Giới thiệu
- AWS WAF (Web Application Firewall) dùng để bảo vệ web application khỏi các **common web exploits** ở **Layer 7**.
- **Layer 7 = HTTP**, nên WAF tập trung vào các kiểu tấn công liên quan đến HTTP.
- Để phân biệt: **Layer 4** là TCP/UDP.

## 1. WAF được triển khai ở đâu?
- WAF có thể gắn vào các dịch vụ sau:
  - **Application Load Balancer (ALB)**
  - **API Gateway**
  - **CloudFront**
  - **AppSync GraphQL API**
  - **Cognito user pools**
- WAF **không thể** triển khai trên **Network Load Balancer (NLB)**.
- Đây là điểm rất dễ bị hỏi bẫy trong kỳ thi AWS.

## 2. Web ACL và các loại rule
- Sau khi triển khai WAF, bạn định nghĩa **Web ACL (Web Access Control List)** và các **rules**.
- Các cách lọc phổ biến:
  - **IP-based rules**: lọc theo địa chỉ IP
    - Có thể tạo **IP sets**
    - Mỗi **IP set** chứa tối đa **10,000 IP addresses**
    - Nếu cần nhiều hơn, có thể dùng nhiều rules
  - Lọc theo:
    - **HTTP headers**
    - **Body**
    - **URI strings**
  - Chống các tấn công phổ biến như:
    - **SQL injection**
    - **Cross-site scripting**
  - **Size constraints**: giới hạn kích thước request, ví dụ chỉ cho phép request đến **2 MB**
  - **Geo match**: cho phép hoặc chặn theo quốc gia
  - **Rate-based rules**:
    - đếm số request theo IP
    - dùng cho **DDoS protection**
    - ví dụ chặn một IP gửi quá **10 requests per second**
- **Rule group** là một tập hợp rules có thể tái sử dụng và gắn vào nhiều Web ACL để tổ chức dễ hơn.

## 3. Phạm vi áp dụng và kiến trúc liên quan
- **Web ACL** là **regional**, ngoại trừ **CloudFront** thì được định nghĩa **globally**.
- WAF là giải pháp **Layer 7**, nên nếu muốn dùng với **ALB** thì phải đi qua ALB, không dùng NLB.
- Vì **ALB không có fixed IP**, transcript đưa ra kiến trúc:
  - Dùng **Global Accelerator** để có **fixed IP**
  - Front **ALB** bằng **Global Accelerator**
  - Gắn **WAF WebACL** tại cùng region với ALB

## 💡 Mẹo ghi nhớ cho kỳ thi AWS
- Nhớ câu then chốt: **WAF = Layer 7 = HTTP**.
- Ghi nhớ các target của WAF: **ALB, API Gateway, CloudFront, AppSync, Cognito**.
- Bẫy thi thường gặp: **WAF không gắn được vào NLB**.
- Nếu đề bài cần **fixed IP + WAF + ALB**, hãy nghĩ đến **Global Accelerator**.
- **CloudFront** là ngoại lệ: Web ACL của WAF được định nghĩa **globally**.

## ✅ Kết luận
- AWS WAF là công cụ bảo vệ **Layer 7** cho các ứng dụng web.
- Nó dùng **Web ACL** và rules để lọc theo IP, headers, body, URI, quốc gia, kích thước request và tốc độ request.
- Với bài toán cần **fixed IP**, có thể kết hợp **Global Accelerator + ALB + WAF** để đạt kiến trúc mong muốn.
