# 361. VMware Cloud on AWS

## 🎯 Giới thiệu
- **VMware Cloud on AWS** là cách mở rộng hạ tầng **VMware Cloud** từ **on-premises data center** sang **AWS**.
- Mục tiêu là giữ nguyên cách quản lý quen thuộc bằng **VMware Cloud software** cho cả data center riêng và môi trường cloud.
- Người dùng có thể tiếp tục vận hành **vSphere-based environment** và các **VMs** trong một mô hình mở rộng sang AWS.

## 1. Khái niệm cốt lõi
- Cho phép **extend** toàn bộ VMware infrastructure lên AWS.
- Có thể chạy các thành phần như:
  - **vSphere**
  - **vSAN**
  - **NSX**
- Ý chính cần nhớ: vẫn quản lý theo hệ sinh thái VMware, nhưng hạ tầng đã được mở rộng sang AWS.

## 2. Use cases chính
- **Mở rộng compute và storage** từ data center lên cloud.
- **Migrate VMware-based workloads** sang AWS.
- Chạy **production workloads** trên nhiều data center.
- Lựa chọn mô hình triển khai:
  - **Private cloud**
  - **Public cloud**
  - **Hybrid cloud**
- Hỗ trợ **disaster recovery strategy** nhờ có thể chuyển nhanh sang cloud bằng cùng bộ công cụ VMware quen thuộc.

## 3. Kết nối với AWS services
- Khi đã ở trên AWS, bạn có thể bắt đầu tận dụng các dịch vụ AWS như:
  - **Amazon EC2**
  - **Amazon FSx**
  - **S3**
  - **RDS**
  - **Direct Connect**
  - **Redshift**
- Điểm quan trọng: VMware Cloud on AWS không chỉ là mở rộng VMware, mà còn giúp tiếp cận hệ sinh thái AWS.

- Nhớ 3 ý chính:
  - **VMware lên AWS**
  - **Giữ nguyên cách quản lý VMware**
  - **Tận dụng AWS services**
- Nếu thấy câu hỏi về:
  - mở rộng **on-premises VMware**
  - chạy **hybrid cloud**
  - cần **disaster recovery**
  - muốn quản lý bằng **VMware Cloud**
  thì rất dễ liên hệ đến **VMware Cloud on AWS**.

## ✅ Kết luận
- **VMware Cloud on AWS** là giải pháp giúp mở rộng hạ tầng VMware từ data center sang AWS mà vẫn giữ mô hình quản trị quen thuộc.
- Dịch vụ này phù hợp cho mở rộng tài nguyên, di chuyển workload, triển khai hybrid, và xây dựng chiến lược disaster recovery.
- Khi lên AWS, bạn còn có thể kết hợp với nhiều dịch vụ AWS khác để mở rộng khả năng hệ thống.
