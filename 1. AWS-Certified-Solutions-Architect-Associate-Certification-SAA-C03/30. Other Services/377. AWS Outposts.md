# 377. AWS Outposts

## 🎯 Giới thiệu
AWS Outposts là giải pháp dành cho **hybrid cloud**, tức doanh nghiệp vừa có **on-premise infrastructure** vừa có **cloud infrastructure**.

- Mục tiêu: đưa trải nghiệm AWS xuống ngay trong **on-premises data center**.
- AWS sẽ triển khai và quản lý các **Outpost racks** trong hạ tầng của doanh nghiệp.
- Các rack này cung cấp **AWS infrastructure, services, API và tools** tương tự như trên cloud.

## 1. Hybrid Cloud và vấn đề Outposts giải quyết
- Trong mô hình hybrid cloud, doanh nghiệp phải xử lý hai hệ thống IT:
  - Một hệ thống dùng **AWS console, CLI, APIs**
  - Một hệ thống riêng cho **on-premises infrastructure**
- Điều này tạo ra:
  - Hai bộ kỹ năng khác nhau
  - Hai loại API khác nhau
  - Sự phức tạp trong vận hành
- **AWS Outposts** ra đời để giảm sự phức tạp này bằng cách mang dịch vụ AWS về on-premises.

## 2. Outposts hoạt động như thế nào
- AWS đến và **set up, manage** các **Outpost racks** trong corporate data center.
- Các server/racks này được triển khai ngay trong **on-premises infrastructure** của doanh nghiệp.
- Bạn có thể dùng các dịch vụ AWS trên chính hạ tầng đó, giống như đang dùng cloud.
- Một điểm khác biệt quan trọng:
  - Nếu **EC2** chạy trên cloud, AWS chịu trách nhiệm về môi trường cloud.
  - Nếu **EC2** chạy trên Outposts trong data center của bạn, bạn phải chịu trách nhiệm về **security** và đặc biệt là **physical security** của rack.

## 3. Lợi ích và dịch vụ hỗ trợ
- Lợi ích chính của Outposts:
  - **Low latency access** đến hệ thống on-premises
  - **Local data processing**
  - **Data residency** trong data center của chính bạn
  - Dễ bắt đầu migration từ **on-premises -> Outposts -> cloud**
  - Là **fully managed service**
- Các dịch vụ có thể chạy trên Outposts theo transcript:
  - **Amazon EC2**
  - **Amazon EBS**
  - **Amazon S3**
  - **Amazon EKS**
  - **Amazon ECS**
  - **Amazon RDS**
  - **Amazon EMR**

## 💡 Mẹo ghi nhớ cho kỳ thi AWS
- Nhớ Outposts là cách AWS “kéo cloud xuống on-premises”.
- Gặp câu hỏi về **hybrid cloud**, nghĩ ngay đến **AWS Outposts**.
- Điểm mấu chốt hay được hỏi:
  - **AWS quản lý rack**
  - **Bạn quản lý physical security tại data center**
  - Dùng để có **low latency** và **data residency**
- Nếu đề bài nói đến việc chạy dịch vụ AWS tại **on-premises data center** với trải nghiệm gần giống cloud, đó rất có thể là **Outposts**.

## ✅ Kết luận
AWS Outposts là giải pháp hybrid cloud giúp doanh nghiệp chạy các dịch vụ AWS ngay trong **on-premises data center** với trải nghiệm, API và tools gần như trên cloud. Đây là lựa chọn quan trọng khi cần **low latency**, **local data processing** và **data residency**, đồng thời vẫn muốn dùng các dịch vụ như **EC2, EBS, S3, EKS, ECS, RDS, EMR**.
