## Nội dung chính: CORS trong Amazon S3

Bài học này giải thích về **CORS — Cross-Origin Resource Sharing**, một cơ chế bảo mật của **web browser**. Đây là nội dung quan trọng và thường xuất hiện trong exam.

---

## 1. CORS là gì?

**CORS** là viết tắt của:

```text
Cross-Origin Resource Sharing
```

Đây là cơ chế bảo mật dựa trên **web browser**, dùng để:

* Cho phép request đến **origin khác**.
* Hoặc từ chối request đến **origin khác**.

Nói đơn giản:

> Khi browser đang truy cập một website, nếu website đó cần gọi tài nguyên từ một website/domain khác, browser sẽ kiểm tra xem origin kia có cho phép hay không thông qua **CORS headers**.

---

## 2. Origin là gì?

Một **origin** được xác định bởi 3 thành phần:

```text
scheme / protocol + host / domain + port
```

Ví dụ:

```text
https://www.example.com
```

Trong URL trên:

| Thành phần            | Giá trị                        |
| --------------------- | ------------------------------ |
| **protocol / scheme** | `https`                        |
| **host / domain**     | `www.example.com`              |
| **port**              | `443` — implied port của HTTPS |

---

## 3. Same Origin là gì?

Hai URL được xem là **same origin** nếu có cùng:

* **scheme / protocol**
* **host / domain**
* **port**

Ví dụ cùng origin:

```text
https://www.example.com/page1
https://www.example.com/page2
```

Ví dụ khác origin:

```text
https://www.example.com
https://other.example.com
```

Dù cùng domain gốc `example.com`, nhưng host khác nhau nên được xem là **different origins**.

---

## 4. Cross-Origin Request hoạt động như thế nào?

Giả sử browser đang truy cập:

```text
https://www.example.com
```

Trang này cần lấy image hoặc asset từ:

```text
https://www.other.com
```

Lúc này, browser sẽ xem đây là **cross-origin request**.

Request sẽ chỉ được thực hiện nếu server đích cho phép thông qua **CORS header**, đặc biệt là:

```text
Access-Control-Allow-Origin
```

---

## 5. Pre-flight Request là gì?

Trước khi thực hiện request thật đến cross-origin server, browser có thể gửi một request kiểm tra trước gọi là:

```text
Pre-flight request
```

Request này thường dùng method:

```text
OPTIONS
```

Browser sẽ hỏi server đích kiểu như:

> Request này đến từ origin `https://www.example.com`, server có cho phép không?

Nếu server đích đã cấu hình **CORS**, nó sẽ trả về các **CORS headers**, ví dụ:

```text
Access-Control-Allow-Origin: https://www.example.com
Access-Control-Allow-Methods: GET, PUT, DELETE
```

Nếu browser thấy CORS headers hợp lệ, nó mới gửi request thật để lấy file, image hoặc gọi API.

---

## 6. CORS Headers quan trọng

Header quan trọng nhất trong bài học là:

```text
Access-Control-Allow-Origin
```

Ý nghĩa:

* Chỉ định origin nào được phép truy cập tài nguyên.
* Có thể cho phép một origin cụ thể.
* Hoặc cho phép tất cả origin bằng wildcard:

```text
*
```

Ví dụ:

```text
Access-Control-Allow-Origin: https://www.example.com
```

hoặc:

```text
Access-Control-Allow-Origin: *
```

---

## 7. CORS áp dụng với Amazon S3 như thế nào?

Trong **Amazon S3**, nếu client/browser thực hiện **cross-origin request** đến S3 bucket, bucket đó cần được cấu hình đúng **CORS headers**.

Đây là một câu hỏi phổ biến trong exam.

Ví dụ tình huống:

* Một S3 bucket dùng để host static website HTML.
* Một S3 bucket khác dùng để lưu assets/images.
* Browser truy cập `index.html` từ bucket đầu tiên.
* Trong `index.html`, có image được lấy từ bucket thứ hai.

Lúc này, request lấy image là **cross-origin request**.

Nếu bucket chứa image không cấu hình đúng **CORS**, browser sẽ chặn request.

---

## 8. Ví dụ với 2 S3 Buckets

Giả sử có 2 bucket:

```text
my-bucket-html
```

Dùng để host static website và chứa file:

```text
index.html
```

Bucket thứ hai:

```text
my-bucket-assets
```

Dùng để chứa:

```text
images/coffee
```

Luồng hoạt động:

1. Browser truy cập static website URL của `my-bucket-html`.
2. S3 trả về file `index.html`.
3. Trong `index.html` có image nằm ở `my-bucket-assets`.
4. Browser gửi request đến `my-bucket-assets` để lấy image.
5. Vì request đến từ origin khác, đây là **cross-origin request**.
6. Nếu `my-bucket-assets` có CORS config phù hợp, image được trả về.
7. Nếu không có CORS config phù hợp, request bị browser chặn.

---

## 9. Điểm cần nhớ về CORS trong S3

CORS trong S3 giúp browser có thể lấy:

* images
* assets
* files
* resources

từ một S3 bucket khác origin với website hiện tại.

Ví dụ:

```text
HTML ở bucket A
Image ở bucket B
```

Nếu browser load HTML từ bucket A nhưng image nằm ở bucket B, bucket B cần cấu hình **CORS** cho phép origin của bucket A.

---

## 10. Tổng kết nhanh

| Keyword                          | Ý nghĩa                                                   |
| -------------------------------- | --------------------------------------------------------- |
| **CORS**                         | Cross-Origin Resource Sharing                             |
| **Origin**                       | scheme/protocol + host/domain + port                      |
| **Same Origin**                  | Cùng protocol, domain, port                               |
| **Cross-Origin Request**         | Request đến origin khác                                   |
| **Pre-flight Request**           | Request kiểm tra trước, thường dùng OPTIONS               |
| **Access-Control-Allow-Origin**  | Header cho phép origin nào được truy cập                  |
| **Access-Control-Allow-Methods** | Header cho phép các method như GET, PUT, DELETE           |
| **Amazon S3 CORS**               | Cấu hình để browser lấy resource từ S3 bucket khác origin |
| `*`                              | Cho phép tất cả origins                                   |
