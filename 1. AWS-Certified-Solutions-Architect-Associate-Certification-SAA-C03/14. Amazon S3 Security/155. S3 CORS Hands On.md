## Nội dung chính: Thực hành CORS trong Amazon S3

Bài học này thực hành cách cấu hình **CORS — Cross-Origin Resource Sharing** giữa 2 **S3 buckets** khi dùng **Static Website Hosting**.

---

## 1. Chỉnh sửa file `index.html` để demo CORS

Trước tiên, cần mở file:

```text
index.html
```

Trong file này có một đoạn code đang bị comment out để demo **CORS**.

Cách bật demo:

* Vào khoảng line 13.
* Xóa phần comment trước thẻ `div`.
* Sau thẻ `script`, xóa phần `--` và dấu mũi tên `>`.

Sau khi bật đoạn code này, trang web đầu tiên sẽ hiển thị:

```text
Hello world
I love coffee
```

và hiển thị hình ảnh coffee.

Ngoài ra, trong `index.html` có một đoạn script dùng `fetch()` để load thêm một file HTML khác:

```text
extra-page.html
```

File này có nội dung:

```text
This extra page has been successfully loaded.
```

---

## 2. Upload `index.html` và `extra-page.html` vào cùng một bucket

Ban đầu, giảng viên upload 2 file vào cùng một **S3 bucket**:

```text
index.html
extra-page.html
```

Sau đó vào phần **Properties** của bucket, tìm **Static Website Endpoint** và mở website.

Kết quả:

* Trang chính hiển thị `Hello world`, `I love coffee`, và hình coffee.
* Nội dung từ `extra-page.html` cũng được load thành công.

Lý do:

> `index.html` và `extra-page.html` đang nằm trong cùng một bucket, tức là cùng **origin**, nên `fetch request` hoạt động bình thường.

Đây là trường hợp **same origin**.

---

## 3. Tạo bucket thứ hai để demo Cross-Origin

Để demo **CORS**, giảng viên tạo một bucket khác tên:

```text
demo-other-origin-stephane
```

Bucket này được đặt ở region khác, ví dụ:

```text
Canada
```

Mục đích là tạo ra một **different origin** / **other origin**.

Khi tạo bucket:

* Unblock all public access.
* Tạo bucket mới.
* Vào **Properties**.
* Enable **Static Website Hosting**.
* Chọn kiểu host là **Host a static website**.
* Đặt **index document** là:

```text
index.html
```

Dù bucket này chưa có `index.html`, cấu hình như vậy vẫn đủ cho demo.

---

## 4. Public bucket thứ hai bằng Bucket Policy

Tiếp theo, cần làm bucket thứ hai public.

Vào:

```text
Permissions → Bucket policy
```

Sau đó copy bucket policy từ bucket cũ, rồi thay phần **Bucket ARN** trong `Resource` thành ARN của bucket mới.

Mục đích:

> Bucket thứ hai cũng có thể public file thông qua static website.

Sau khi save policy, bucket này có thể phục vụ file public.

---

## 5. Upload `extra-page.html` vào bucket thứ hai

Giảng viên upload file:

```text
extra-page.html
```

vào bucket thứ hai.

Sau đó mở **Object URL** hoặc static website URL của file.

Kết quả:

```text
This extra page has been successfully loaded.
```

Điều này chứng minh file `extra-page.html` trong bucket thứ hai đã public và truy cập được.

---

## 6. Xóa `extra-page.html` khỏi bucket chính

Sau khi đã đưa `extra-page.html` sang bucket thứ hai, giảng viên xóa `extra-page.html` khỏi bucket đầu tiên.

Khi refresh website ở bucket đầu tiên, trình duyệt báo:

```text
404 Not Found
```

Lý do:

> `index.html` vẫn đang tìm `extra-page.html` trong cùng bucket, nhưng file đó đã bị xóa.

Điều này cho thấy cần sửa `index.html` để trỏ sang file ở bucket thứ hai.

---

## 7. Sửa `index.html` để fetch file từ other origin

Tiếp theo, giảng viên lấy full URL của `extra-page.html` trong bucket thứ hai, ví dụ:

```text
http://<other-origin-bucket-static-website-endpoint>/extra-page.html
```

Sau đó sửa `index.html`.

Thay vì fetch:

```text
extra-page.html
```

thì fetch full path đến file ở bucket khác:

```text
http://<other-origin-bucket-static-website-endpoint>/extra-page.html
```

Sau đó upload lại `index.html` vào bucket đầu tiên và overwrite file cũ.

Lúc này, browser sẽ tạo một:

```text
Cross-Origin Resource Sharing request
```

vì website đang chạy từ bucket đầu tiên nhưng gọi file từ bucket thứ hai.

---

## 8. Lỗi CORS khi chưa cấu hình CORS settings

Sau khi upload lại `index.html`, giảng viên mở website bucket đầu tiên và mở:

```text
Chrome Developer Tools
```

hoặc:

```text
Web Developer Tools
```

Sau đó refresh page.

Trong **Console**, browser báo lỗi:

```text
Cross-Origin Request Blocked
```

Nguyên nhân:

> Request đang đi từ bucket đầu tiên sang bucket thứ hai, nhưng bucket thứ hai chưa được cấu hình **CORS**.

Lỗi xảy ra vì thiếu CORS header:

```text
Access-Control-Allow-Origin
```

Nói cách khác:

> Browser chặn request vì **other origin bucket** chưa cho phép origin của bucket đầu tiên truy cập.

---

## 9. Thêm CORS settings cho bucket thứ hai

Để fix lỗi, vào bucket thứ hai:

```text
Permissions → CORS
```

Sau đó edit CORS settings bằng JSON.

Ví dụ cấu hình:

```json
[
  {
    "AllowedHeaders": ["*"],
    "AllowedMethods": ["GET"],
    "AllowedOrigins": ["http://<first-bucket-static-website-endpoint>"],
    "ExposeHeaders": []
  }
]
```

Trong đó:

* `AllowedOrigins` là URL của bucket đầu tiên.
* URL nên dùng `http`.
* Không thêm dấu `/` ở cuối URL.
* `AllowedMethods` có `GET` vì browser đang fetch file HTML.

Ví dụ đúng:

```text
http://demo-origin-stephane.s3-website-region.amazonaws.com
```

Ví dụ sai:

```text
http://demo-origin-stephane.s3-website-region.amazonaws.com/
```

---

## 10. Kiểm tra sau khi thêm CORS

Sau khi save **CORS settings**, refresh lại website bucket đầu tiên.

Kết quả:

```text
This extra page has been successfully loaded.
```

Điều này chứng minh:

> Bucket thứ hai đã cho phép request từ bucket đầu tiên thông qua **CORS headers**.

---

## 11. Kiểm tra CORS headers trong Network tab

Trong **Chrome Developer Tools**, mở tab:

```text
Network
```

Chọn request đến:

```text
extra-page.html
```

Trong phần **Response Headers**, có thể thấy:

```text
Access-Control-Allow-Methods: GET
```

và:

```text
Access-Control-Allow-Origin: http://<first-bucket-static-website-endpoint>
```

Đây là các **CORS headers** cho phép browser thực hiện **cross-origin request**.

---

## Tổng kết nhanh

| Keyword                          | Ý nghĩa                                                  |
| -------------------------------- | -------------------------------------------------------- |
| **CORS**                         | Cross-Origin Resource Sharing                            |
| **Same Origin**                  | `index.html` và `extra-page.html` nằm cùng bucket/origin |
| **Other Origin**                 | Resource nằm ở bucket khác origin                        |
| **Static Website Hosting**       | Cho phép S3 bucket hoạt động như website tĩnh            |
| **fetch()**                      | JavaScript function dùng để gọi resource                 |
| **Chrome Developer Tools**       | Công cụ kiểm tra lỗi trên browser                        |
| **Cross-Origin Request Blocked** | Browser chặn request vì thiếu CORS                       |
| **Access-Control-Allow-Origin**  | Header cho phép origin được truy cập resource            |
| **Access-Control-Allow-Methods** | Header khai báo method được phép, ví dụ `GET`            |
| **AllowedOrigins**               | Danh sách origin được phép gọi resource                  |
| **AllowedMethods**               | Danh sách HTTP methods được phép                         |

---

## Ghi nhớ cho exam

```text
CORS must be configured on the target bucket.
```

```text
If Bucket A hosts index.html and fetches files from Bucket B, Bucket B must allow Bucket A in CORS settings.
```

Nói ngắn gọn:

> Khi browser load website từ **S3 Bucket A** nhưng dùng `fetch()` để lấy file từ **S3 Bucket B**, thì **Bucket B** phải cấu hình **CORS** cho phép origin của **Bucket A**. Nếu không, browser sẽ báo **Cross-Origin Request Blocked**.
