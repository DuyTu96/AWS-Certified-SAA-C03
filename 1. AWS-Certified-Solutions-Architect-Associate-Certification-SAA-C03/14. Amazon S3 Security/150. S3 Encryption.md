## 1. Các phương pháp mã hóa object trong Amazon S3

Amazon S3 hỗ trợ 4 cách mã hóa chính:

### 1. SSE-S3 — Server-Side Encryption with Amazon S3-managed keys

Đây là cơ chế mã hóa phía server, trong đó key được **AWS quản lý, sở hữu và xử lý**.

Đặc điểm chính:

* AWS tự quản lý encryption key.
* Người dùng không có quyền truy cập key.
* Sử dụng chuẩn mã hóa **AES-256**.
* Cần header:

```http
x-amz-server-side-encryption: AES256
```

* Hiện nay **SSE-S3 được bật mặc định** cho bucket và object mới.

Hiểu đơn giản: user upload file lên S3, Amazon S3 dùng **S3-owned key** để mã hóa object rồi lưu vào bucket.

---

### 2. SSE-KMS — Server-Side Encryption with AWS KMS keys

SSE-KMS cũng là mã hóa phía server, nhưng key được quản lý thông qua **AWS KMS — Key Management Service**.

Ưu điểm:

* Người dùng có thể kiểm soát key tốt hơn.
* Có thể quản lý, phân quyền, audit việc sử dụng key.
* Việc sử dụng key được ghi log trong **AWS CloudTrail**.

Header cần dùng:

```http
x-amz-server-side-encryption: aws:kms
```

Khi đọc object được mã hóa bằng SSE-KMS, user cần có:

* Quyền truy cập object trong S3.
* Quyền sử dụng **KMS key** đã dùng để mã hóa object.

Điểm cần nhớ cho exam:

* SSE-KMS tạo thêm một lớp bảo mật.
* Mỗi lần upload/download object có dùng KMS key sẽ gọi API của KMS, ví dụ:

  * `GenerateDataKey`
  * `Decrypt`
* Các API call này bị giới hạn bởi **KMS quotas** theo region, thường khoảng **5,000 đến 30,000 requests/second**.
* Với S3 bucket có throughput rất cao, SSE-KMS có thể gặp giới hạn quota nếu gọi KMS quá nhiều.

---

### 3. SSE-C — Server-Side Encryption with Customer-provided keys

SSE-C là mã hóa phía server, nhưng encryption key do **customer tự cung cấp**.

Đặc điểm:

* Key được quản lý bên ngoài AWS.
* Amazon S3 dùng key do client gửi lên để mã hóa/giải mã object.
* Amazon S3 **không lưu trữ encryption key**.
* Sau khi dùng xong, key sẽ bị loại bỏ.
* Bắt buộc sử dụng **HTTPS**.
* Key phải được gửi trong **HTTP headers** ở mỗi request.

Khi muốn đọc lại object, user phải cung cấp đúng key đã dùng để mã hóa object đó.

---

### 4. Client-Side Encryption

Với client-side encryption, dữ liệu được mã hóa **trước khi gửi lên Amazon S3**.

Đặc điểm:

* Client tự mã hóa dữ liệu trước khi upload.
* Amazon S3 chỉ lưu file đã được mã hóa.
* Khi download về, client tự giải mã dữ liệu.
* Client hoàn toàn quản lý:

  * encryption key
  * quá trình encryption
  * quá trình decryption

Có thể dùng thư viện như **Client-Side Encryption Library** để triển khai dễ hơn.

---

## 2. Encryption in Transit / Encryption in Flight

Ngoài mã hóa object khi lưu trữ, bài học còn nói về mã hóa dữ liệu khi truyền tải.

Encryption in transit còn được gọi là:

* **SSL**
* **TLS**
* **Encryption in flight**

Amazon S3 có 2 loại endpoint:

| Endpoint | Mã hóa       |
| -------- | ------------ |
| HTTP     | Không mã hóa |
| HTTPS    | Có mã hóa    |

Khuyến nghị luôn dùng **HTTPS** để truyền dữ liệu an toàn.

Đặc biệt:

* Nếu dùng **SSE-C**, bắt buộc phải dùng **HTTPS** vì encryption key được gửi trong HTTP headers.

---

## 3. Cách bắt buộc dùng HTTPS với S3 Bucket Policy

Có thể ép buộc mọi request đến S3 bucket phải dùng HTTPS bằng **bucket policy**.

Ý tưởng chính:

* Deny request nếu điều kiện:

```json
"aws:SecureTransport": "false"
```

Ý nghĩa:

* `aws:SecureTransport = true`: request dùng HTTPS.
* `aws:SecureTransport = false`: request dùng HTTP, không an toàn.

Vì vậy, policy sẽ chặn mọi request không dùng HTTPS.

---

## 4. Bảng so sánh nhanh

| Cơ chế                     | Ai quản lý key? | Mã hóa ở đâu? | Ghi chú quan trọng                         |
| -------------------------- | --------------- | ------------- | ------------------------------------------ |
| **SSE-S3**                 | AWS / Amazon S3 | Server-side   | Dùng AES-256, bật mặc định                 |
| **SSE-KMS**                | AWS KMS         | Server-side   | Có audit qua CloudTrail, cần quyền KMS key |
| **SSE-C**                  | Customer        | Server-side   | S3 không lưu key, bắt buộc HTTPS           |
| **Client-Side Encryption** | Client          | Client-side   | Client tự mã hóa trước khi upload          |

---

## Nội dung cần nhớ nhất

* **SSE-S3**: đơn giản nhất, AWS quản lý key, dùng `AES256`.
* **SSE-KMS**: dùng AWS KMS, có kiểm soát key và audit qua CloudTrail, nhưng có thể bị giới hạn bởi **KMS API quotas**.
* **SSE-C**: customer tự cung cấp key, S3 không lưu key, bắt buộc dùng **HTTPS**.
* **Client-Side Encryption**: client tự mã hóa và giải mã, S3 chỉ lưu dữ liệu đã mã hóa.
* **Encryption in transit** nên dùng **HTTPS**.
* Có thể dùng **bucket policy** với `aws:SecureTransport` để bắt buộc request phải dùng HTTPS.
