## Nội dung chính: Thực hành MFA Delete trong Amazon S3

Bài học này demo cách bật và kiểm tra **MFA Delete** cho một **S3 bucket** đã bật **Versioning**.

---

## 1. Tạo S3 bucket và bật Bucket Versioning

Giảng viên tạo một bucket mới, ví dụ:

```text
demo-stephane-MFA-delete-2020
```

Bucket được tạo ở region:

```text
eu-west-1
```

Khi tạo bucket, cần bật:

```text
Bucket Versioning
```

Sau khi tạo bucket xong, vào:

```text
Properties → Bucket Versioning → Edit
```

Tại đây có thể thấy:

```text
Multi-Factor Authentication (MFA) Delete = Disabled
```

Điểm quan trọng:

> Không thể bật **MFA Delete** trực tiếp bằng **Amazon S3 Console UI**.

Muốn enable **MFA Delete**, phải dùng:

```text
AWS CLI
```

---

## 2. Điều kiện trước khi bật MFA Delete

Trước khi thực hành, cần đảm bảo đã setup **MFA device** cho:

```text
Root account
```

Trong bài học, giảng viên đang login bằng **root account**, sau đó vào:

```text
Security credentials
```

Trong phần:

```text
Multi-Factor Authentication (MFA)
```

cần có sẵn một **virtual MFA device** hoặc thiết bị MFA khác.

Tại đây sẽ có:

```text
MFA device ARN
```

ARN này sẽ được dùng trong câu lệnh **AWS CLI** để enable hoặc disable **MFA Delete**.

---

## 3. Cấu hình AWS CLI bằng root account

Để bật **MFA Delete**, giảng viên tạo **access key** cho root account và cấu hình **AWS CLI profile**.

Lệnh cấu hình:

```bash
aws configure --profile root-MFA-delete-demo
```

Thông tin cần nhập:

```text
Access Key ID
Secret Access Key
Default region name: eu-west-1
```

Sau đó kiểm tra profile hoạt động bằng lệnh:

```bash
aws s3 ls --profile root-MFA-delete-demo
```

Nếu lệnh trả về danh sách buckets, nghĩa là profile đã được cấu hình đúng.

Lưu ý bảo mật rất quan trọng:

> Không nên dùng **root access key** trừ trường hợp đặc biệt như enable **MFA Delete**.
> Không bao giờ chia sẻ **root access key** hoặc **secret access key**.

---

## 4. Enable MFA Delete bằng AWS CLI

Để bật **MFA Delete**, dùng lệnh dạng:

```bash
aws s3api put-bucket-versioning \
  --bucket demo-stephane-MFA-delete-2020 \
  --versioning-configuration Status=Enabled,MFADelete=Enabled \
  --mfa "arn-of-mfa-device mfa-code" \
  --profile root-MFA-delete-demo
```

Trong đó:

* `--bucket`: tên bucket cần bật MFA Delete.
* `Status=Enabled`: bật **Versioning**.
* `MFADelete=Enabled`: bật **MFA Delete**.
* `--mfa`: gồm **MFA device ARN** và **MFA code** hiện tại.
* `--profile`: profile root account đã cấu hình.

Ví dụ phần `--mfa` có dạng:

```text
arn:aws:iam::<account-id>:mfa/root-account-mfa-device 123456
```

Nếu MFA code sai hoặc hết hạn, command sẽ fail. Cần lấy code mới từ app MFA như **Google Authenticator** rồi chạy lại.

---

## 5. Kiểm tra MFA Delete đã được bật

Sau khi chạy lệnh thành công, quay lại bucket trong Console:

```text
Properties → Bucket Versioning
```

Kết quả sẽ thấy:

```text
Bucket Versioning = Enabled
Multi-Factor Authentication (MFA) Delete = Enabled
```

Điều này xác nhận **MFA Delete** đã được bật thành công.

---

## 6. Kiểm tra behavior khi xóa object

Giảng viên upload một file, ví dụ:

```text
coffee.jpg
```

Sau đó xóa object trong Console.

Vì bucket đã bật **Versioning**, thao tác delete bình thường sẽ không xóa vĩnh viễn object, mà chỉ tạo:

```text
Delete marker
```

Điều này vẫn hoạt động bình thường, vì đây chưa phải là permanent delete.

---

## 7. MFA Delete chặn Permanent Delete

Sau khi bật hiển thị version, bucket có thể có:

* Object version thật
* Delete marker

Nếu cố gắng xóa một **specific version ID** trong Console, đây là thao tác:

```text
Permanent delete
```

Khi đó S3 sẽ chặn và báo rằng không thể delete object vì:

```text
Multi-Factor Authentication (MFA) Delete is enabled for this bucket
```

Ý nghĩa:

> Khi **MFA Delete** đang enabled, bạn không thể xóa vĩnh viễn một object version nếu không dùng MFA qua CLI.

---

## 8. Disable MFA Delete bằng AWS CLI

Để tắt **MFA Delete**, cũng phải dùng **AWS CLI** với root account và MFA code.

Lệnh tương tự như lúc enable, nhưng đổi:

```text
MFADelete=Disabled
```

Ví dụ:

```bash
aws s3api put-bucket-versioning \
  --bucket demo-stephane-MFA-delete-2020 \
  --versioning-configuration Status=Enabled,MFADelete=Disabled \
  --mfa "arn-of-mfa-device mfa-code" \
  --profile root-MFA-delete-demo
```

Sau khi chạy thành công, quay lại Console kiểm tra:

```text
Multi-Factor Authentication (MFA) Delete = Disabled
```

Lúc này có thể xóa permanent delete object version hoặc delete marker trong Console.

---

## 9. Xóa Root Access Key sau khi thực hành

Cuối bài, giảng viên nhấn mạnh cần xóa hoặc deactivate **root access key** sau khi dùng xong.

Lý do:

> Việc giữ **root access key** là rủi ro bảo mật rất lớn.

Nên thực hiện:

```text
Deactivate root access key
Delete root access key
```

Sau khi hoàn thành thao tác enable/disable **MFA Delete**, không nên giữ root access key tồn tại.

---

## Tổng kết nhanh

| Keyword                  | Ý nghĩa                                               |
| ------------------------ | ----------------------------------------------------- |
| **MFA Delete**           | Bảo vệ việc xóa vĩnh viễn object version              |
| **MFA**                  | Multi-Factor Authentication                           |
| **Bucket Versioning**    | Bắt buộc phải enabled trước khi dùng MFA Delete       |
| **AWS CLI**              | Công cụ cần dùng để enable/disable MFA Delete         |
| **Amazon S3 Console UI** | Không bật được MFA Delete trực tiếp                   |
| **Root account**         | Chỉ root account mới enable/disable MFA Delete        |
| **MFA device ARN**       | ARN của thiết bị MFA dùng trong CLI                   |
| **MFA code**             | Mã xác thực lấy từ MFA device/app                     |
| **Delete marker**        | Được tạo khi delete object trong bucket có Versioning |
| **Permanent delete**     | Xóa vĩnh viễn một specific object version             |
| **Root access key**      | Nên xóa/deactivate sau khi thực hành                  |

## Ghi nhớ cho exam

```text
MFA Delete requires Versioning to be enabled.
```

```text
MFA Delete can only be enabled or disabled by the root account.
```

```text
MFA Delete cannot be enabled from the S3 Console UI; use AWS CLI.
```

```text
MFA Delete protects against permanent deletion of object versions.
```

Nói ngắn gọn:

> **MFA Delete** là lớp bảo vệ bổ sung cho **S3 Versioning**, yêu cầu **MFA code** khi muốn xóa vĩnh viễn object version hoặc thay đổi trạng thái MFA Delete. Tính năng này phải được bật/tắt bằng **AWS CLI** thông qua **root account**.
