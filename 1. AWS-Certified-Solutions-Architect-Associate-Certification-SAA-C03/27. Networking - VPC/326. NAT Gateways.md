# 326. NAT Gateways

## 🎯 Giới thiệu
NAT Gateway là cách tốt hơn để cho **private instances trong private subnet** truy cập Internet mà không cần public IP.  
Điểm nổi bật trong transcript:

- Là **managed service** → không cần tự quản trị
- **High availability** tốt hơn NAT instance
- **Bandwidth** cao hơn
- Không cần quản lý **security groups**
- Được tạo trong một **specific AZ**
- Có **Elastic IP / public IP**
- Tính phí theo **giờ sử dụng** và **data transfer**

## 1. NAT Gateway hoạt động như thế nào 🛣️
NAT Gateway thường được đặt trong **public subnet**, rồi private subnet sẽ route traffic đi qua nó để ra Internet.

- **Private subnet** không truy cập Internet trực tiếp
- **Public subnet** đã kết nối với **Internet Gateway**
- NAT Gateway nhờ đó có Internet connectivity
- Sau đó chỉnh **route table** của private subnet để trỏ traffic ra NAT Gateway

```mermaid
flowchart LR
    A[Instance trong private subnet] --> B[Route table của private subnet]
    B --> C[NAT Gateway trong public subnet]
    C --> D[Internet Gateway]
    D --> E[Internet]
```

## 2. High Availability và AZ 🔁
NAT Gateway có tính **resilience trong phạm vi một Availability Zone (AZ)**.

- Redundant trong **single AZ**
- Nếu muốn **fault tolerance across AZ**
  - cần tạo **multiple NAT Gateways**
  - mỗi NAT Gateway ở một AZ khác nhau
- Network traffic được **confine trong từng AZ**
- Nếu một AZ down, AZ khác vẫn hoạt động nếu có NAT Gateway riêng

## 3. So sánh với NAT Instance ⚖️
Transcript nhấn mạnh NAT Gateway khác NAT instance ở nhiều điểm:

- **NAT Gateway**
  - Managed service
  - Không cần quản lý software/patching
  - Không dùng **security groups**
  - Không thể dùng làm **bastion host**
  - Bandwidth lên tới **5 Gbps**, tự động scale đến **100 Gbps**
  - Tính phí theo giờ + data transfer

- **NAT Instance**
  - Do bạn tự quản lý
  - Phải lo **software**, **patches**
  - Bandwidth phụ thuộc **instance type**
  - Cần cấu hình **security groups**
  - Có thể dùng làm **bastion host**
  - Có thể cần script để quản lý failover

## 💡 Mẹo ghi nhớ cho kỳ thi AWS
- **NAT Gateway = private subnet ra Internet qua public subnet**
- Muốn ra Internet thì phải có **Internet Gateway**
- **NAT Gateway không cần security groups**
- **NAT Gateway chỉ HA trong 1 AZ**; muốn chịu lỗi nhiều AZ thì tạo nhiều NAT Gateway
- Nếu đề hỏi:
  - cần **managed**, **high throughput**, **ít quản trị** → chọn **NAT Gateway**
  - cần tự quản lý, có thể làm **bastion host** → nghĩ đến **NAT Instance**
- Ghi nhớ luồng chuẩn: **private subnet → NAT Gateway → Internet Gateway → Internet**

## ✅ Kết luận
NAT Gateway là giải pháp phù hợp khi cần cho **private subnet** truy cập Internet với mô hình **managed**, dễ vận hành, bandwidth cao và không phải quản lý security groups.  
Trong đề thi, hãy chú ý các từ khóa: **public subnet**, **private subnet**, **Internet Gateway**, **single AZ HA**, và **no security groups**.
