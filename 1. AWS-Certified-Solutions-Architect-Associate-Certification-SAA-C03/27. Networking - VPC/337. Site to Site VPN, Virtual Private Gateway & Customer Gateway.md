# 337. Site to Site VPN, Virtual Private Gateway & Customer Gateway

## 🎯 Giới thiệu
- **Site-to-site VPN** dùng để kết nối **VPC** với **corporate data center** thông qua **public internet** nhưng vẫn được **encrypt**.
- Mô hình này tạo ra liên kết giữa mạng AWS và mạng on-premises bằng:
  - **Virtual Private Gateway (VGW)** ở phía AWS
  - **Customer Gateway (CGW)** ở phía doanh nghiệp

```mermaid
flowchart TD
  DC[Corporate Data Center] --> CGW[Customer Gateway (CGW)]
  CGW <-- encrypted VPN over public internet --> VGW[Virtual Private Gateway (VGW)]
  VGW --> VPC[VPC / Subnets]
```

## 1. Thành phần chính của Site-to-Site VPN
- **VGW (Virtual Private Gateway)**:
  - Là **VPN concentrator** ở phía AWS.
  - Được **create** và **attach** vào **VPC** mà bạn muốn kết nối.
  - Có thể **customize ASN number**.
- **CGW (Customer Gateway)**:
  - Là **software** hoặc **physical device** ở phía data center.
  - Là đầu mối phía on-premises của kết nối VPN.
- Kết nối này là **private site-to-site VPN connection** nhưng **đi qua public internet**.

## 2. Cách chọn IP cho Customer Gateway
- Nếu **customer gateway public**:
  - Dùng **public internet-routable IP address** của CGW.
- Nếu **customer gateway private**:
  - Thường nằm **behind a NAT device** có **NAT-T enabled**.
  - Khi đó, IP cần dùng cho CGW là **public IP của NAT device**.
- Đây là điểm dễ xuất hiện trong đề thi AWS.

## 3. Các điểm cần nhớ cho kỳ thi
- **Route propagation** phải được enable trong **VPC route tables** thì site-to-site VPN mới hoạt động.
- Nếu muốn **ping EC2 instances** từ on-premises:
  - Cần bật **ICMP** inbound trong **Security Group**.
- **AWS VPN CloudHub**:
  - Dùng khi có **multiple customer networks / multiple data centers**.
  - Mục tiêu là cho các site này **communicate with one another** thông qua **multiple VPN connections**.
  - Đây là mô hình **hub-and-spoke** chi phí thấp, chỉ dùng **VPN**.
  - Thiết lập bằng cách:
    - Tạo nhiều **site-to-site VPN connections** trên cùng một **VGW**
    - Bật **dynamic routing**
    - Cấu hình **route tables**

## 📊 Bảng tóm tắt
| Tiêu chí | Mô tả |
|----------|------|
| Mục tiêu | Kết nối VPC với corporate data center qua VPN |
| AWS side | **Virtual Private Gateway (VGW)** |
| On-premises side | **Customer Gateway (CGW)** |
| Đường truyền | **Public internet** nhưng được **encrypt** |
| Điểm thi hay hỏi | Chọn IP cho CGW, **route propagation**, ICMP inbound cho EC2 |
| Mở rộng nhiều site | **AWS VPN CloudHub** |

## 💡 Mẹo ghi nhớ cho kỳ thi AWS
- **VGW = AWS side**, **CGW = customer side**.
- **VPN có encrypt** nhưng vẫn **đi qua public internet**.
- Nếu VPN đã tạo mà chưa hoạt động, hãy nghĩ ngay đến **route propagation**.
- Muốn ping **EC2** từ on-premises, đừng quên **ICMP inbound rule** trong **Security Group**.
- **CloudHub** = nhiều site, nhiều VPN, cùng một **VGW**.

## ✅ Kết luận
- **Site-to-site VPN** là cách kết nối **VPC** với **data center** bằng VPN encrypted trên public internet.
- Hai thành phần cốt lõi là **VGW** và **CGW**.
- Các điểm quan trọng để ôn thi là **IP của CGW**, **route propagation**, **ICMP inbound**, và **AWS VPN CloudHub**.
