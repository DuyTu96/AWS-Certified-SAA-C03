# 322. Bastion Hosts

## 🎯 Giới thiệu
- **Bastion host** là một **EC2 instance** dùng để truy cập vào các **EC2 instance** nằm trong **private subnet**.
- Bối cảnh bài học: người dùng ở **public internet** không thể kết nối trực tiếp đến EC2 trong private subnet.
- Bastion host là một cách để thực hiện **SSH** vào các máy EC2 private một cách gián tiếp.

## 1. Bastion Host là gì?
- Bastion host là một **EC2 instance** được đặt tên là bastion host.
- Instance này phải nằm trong **public subnet**.
- Nó đóng vai trò trung gian để truy cập các EC2 instance trong **private subnet**.
- Có thể dùng để truy cập:
  - 1 EC2 instance private
  - hoặc nhiều EC2 instances private

## 2. Luồng kết nối SSH 🔗
- Người dùng kết nối từ máy cá nhân trên internet đến **bastion host** bằng **SSH**.
- Từ bastion host, tiếp tục **SSH** đến EC2 instance trong private subnet.
- Bastion host có thể truy cập EC2 private vì cả hai đều nằm trong cùng **VPC**.

```mermaid
flowchart TD
    U[User / Computer on public internet] -->|SSH| B[Bastion Host EC2<br/>in public subnet]
    B -->|SSH| P[EC2 instance(s)<br/>in private subnet]
```

## 3. Security Group Rules 🔐
- Với **bastion host security group**:
  - phải cho phép truy cập từ internet
  - nhưng nên **giới hạn tối đa** thay vì mở rộng cho tất cả
  - ví dụ: chỉ cho phép từ **public CIDR** của công ty hoặc các IP truy cập hợp lệ
- Lý do:
  - nếu một attacker ngẫu nhiên truy cập được bastion host, đó sẽ là rủi ro bảo mật cho toàn bộ hạ tầng
- Với **security group** của EC2 trong private subnet:
  - phải cho phép **SSH** trên **port 22**
  - nguồn truy cập là **private IP của bastion host** hoặc **security group của bastion host**
  - hai cách này được xem là tương đương trong ngữ cảnh bài học
- Điểm cần nhớ: traffic vào private EC2 được xem là **xuất phát từ bastion host**

## 📊 Bảng tóm tắt
| Tiêu chí | Mô tả |
|----------|------|
| Mục đích | SSH vào EC2 trong private subnet |
| Vị trí của bastion host | public subnet |
| Kết nối từ người dùng | SSH vào bastion host |
| Kết nối tiếp theo | SSH từ bastion host vào EC2 private |
| Security group của bastion host | Cho phép truy cập từ internet nhưng phải giới hạn IP/CIDR |
| Security group của EC2 private | Cho phép SSH port 22 từ private IP hoặc security group của bastion host |
| Ý nghĩa bảo mật | Hạn chế rủi ro nếu bastion host bị truy cập trái phép |

## 💡 Mẹo ghi nhớ cho kỳ thi AWS
- **Bastion host = cửa trung gian vào private subnet**.
- Nhớ 2 bước:
  - **User -> Bastion host**
  - **Bastion host -> Private EC2**
- Nhớ vị trí:
  - bastion host ở **public subnet**
  - EC2 đích ở **private subnet**
- Nhớ rule quan trọng:
  - bastion host mở cho internet nhưng phải **restrict IP/CIDR**
  - private EC2 cho phép **SSH port 22** từ bastion host
- Nếu thấy câu hỏi về truy cập EC2 private từ internet, hãy nghĩ ngay đến **bastion host**.

## ✅ Kết luận
- Bastion host là một **EC2 instance trong public subnet** dùng để **SSH** vào các EC2 trong **private subnet**.
- Đây là mô hình trung gian giúp truy cập an toàn hơn thay vì mở trực tiếp private EC2 ra internet.
- Trọng tâm thi: hiểu **luồng SSH** và **security group rules** cho cả bastion host lẫn EC2 private.
