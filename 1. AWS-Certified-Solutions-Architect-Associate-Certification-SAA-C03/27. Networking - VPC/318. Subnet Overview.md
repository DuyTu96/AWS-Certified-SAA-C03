# 318. Subnet Overview

## 🎯 Giới thiệu
- Trong VPC, bước tiếp theo là tạo **subnet**.
- Bài giảng tạo 2 subnet:
  - **public subnet**
  - **private subnet**
- Cả hai được đặt trong **một Availability Zone**.
- Mục tiêu là hiểu:
  - subnet là gì
  - vì sao một subnet có thể public hoặc private
  - AWS giữ lại những IP nào trong subnet

## 1. Khái niệm subnet 🧩
- **Subnet** là một **sub-range của IPv4 addresses** trong VPC.
- Mỗi subnet là một phần nhỏ của CIDR block trong VPC.
- EC2 instance chỉ có thể dùng các IP **không bị AWS reserved** trong subnet.

## 2. Các IP bị AWS reserved trong subnet 🔒
- AWS luôn reserve **5 IP addresses** trong mỗi subnet:
  - **4 IP đầu tiên**
  - **1 IP cuối cùng**
- Các IP này:
  - không dùng được
  - không gán cho **EC2 instance** được

### Ví dụ với `10.0.0.0/24`
- `10.0.0.0`: **network address**
- `10.0.0.1`: reserved cho **VPC router**
- `10.0.0.2`: reserved cho **Amazon-provided DNS**
- `10.0.0.3`: **reserved for future use**
- `10.0.0.255`: **network broadcast address**
  - AWS không hỗ trợ broadcast trong VPC, nên địa chỉ này cũng bị reserved

## 3. Chọn size subnet cho exam 📘
- Nếu cần **29 IP usable** cho EC2 instance:
  - **/27** không đủ
  - /27 có **32 IP**
  - trừ 5 IP reserved còn **27 usable**
- Cần chọn **/26**
  - /26 có **64 IP**
  - trừ 5 IP reserved còn **59 usable**
  - đủ cho nhu cầu **29 IP**

## 📊 Bảng tóm tắt
| Tiêu chí | Mô tả |
|----------|------|
| Subnet | Sub-range của IPv4 addresses trong VPC |
| Số subnet trong bài | 2 subnet: **public** và **private** |
| Availability Zone | Cả hai subnet nằm trong **1 AZ** |
| IP reserved | AWS reserve **5 IP/subnet** |
| IP không dùng được | 4 IP đầu + 1 IP cuối |
| Ví dụ `10.0.0.0/24` | `.0` network, `.1` VPC router, `.2` Amazon DNS, `.3` future use, `.255` broadcast |
| Exam tip | Cần 29 IP thì **/27 không đủ**, **/26 đủ** |

## 💡 Mẹo ghi nhớ cho kỳ thi AWS
- Nhớ công thức: **usable IPs = total IPs - 5 reserved**
- Nếu đề bài hỏi số IP cần dùng cho EC2, hãy kiểm tra nhanh:
  - **/27 = 32 - 5 = 27 usable**
  - **/26 = 64 - 5 = 59 usable**
- AWS luôn reserved **5 IP** trong mỗi subnet, đây là điểm rất dễ ra thi.

## ✅ Kết luận
- Subnet là một phần của IPv4 range trong VPC.
- AWS reserve **5 IP** trong mỗi subnet, nên số IP usable luôn ít hơn số IP tổng.
- Khi làm bài thi, hãy tính nhanh số IP usable để chọn đúng kích thước subnet như **/26** thay vì **/27** khi cần nhiều IP hơn.
