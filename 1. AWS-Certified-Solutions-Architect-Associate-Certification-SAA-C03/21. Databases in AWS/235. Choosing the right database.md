# 235. Choosing the right database

## 🎯 Giới thiệu
Trong phần này, mục tiêu chính là học cách **chọn đúng database cho đúng workload** trong kỳ thi AWS. AWS có rất nhiều managed databases, nên đáp án thường phụ thuộc vào **đặc điểm kiến trúc và yêu cầu của đề bài**, không phải chỉ nhớ tên dịch vụ.

Các câu hỏi thường xoay quanh:
- Workload **write-heavy**, **read-heavy** hay **balanced**
- Dữ liệu có **thay đổi/fluctuate** trong ngày không
- Khối lượng dữ liệu, thời gian lưu trữ, và tốc độ tăng trưởng
- Kích thước object trung bình
- Tần suất truy cập và cách truy cập dữ liệu
- **Data durability**
- **Source of truth**
- **Latency requirements**
- Số lượng **concurrent users**
- **Data model**, cách query, có cần **joins** không
- Dữ liệu **structured** hay **semi-structured**
- Có cần **strong schema** hay cần linh hoạt hơn
- Có cần **reporting**, **search**, hoặc database quan hệ / **NoSQL**
- Có **license cost** không
- Có muốn chuyển sang **Cloud Native databases** như **Aurora** không

## 1. RDBMS databases
Đây là nhóm database dùng khi:
- Cần **SQL**
- Cần **online transaction processing**
- Cần **joins**

Các dịch vụ được nhắc đến:
- **RDS**
- **Aurora**

Điểm chính:
- Phù hợp với bài toán quan hệ
- Có lợi thế khi đề bài nhắc đến **SQL** và **joins**

## 2. NoSQL, object store và search
### NoSQL databases
Nhìn chung:
- **Flexible** hơn
- Thường không có **join capabilities**
- Thường không dùng **SQL query language** theo cách truyền thống

Các dịch vụ:
- **DynamoDB**
- **ElastiCache**
- **Neptune**
- **DocumentDB**
- **Keyspaces**

### Object store
Dùng để:
- Lưu **big objects**
- Lưu **backups** và **archives**

Dịch vụ được nhắc đến:
- **Amazon S3**
- **Glacier**

### Search databases
Dùng cho:
- **Free text**
- **Unstructured searches**

Dịch vụ:
- **OpenSearch**

## 3. Analytics, graph, ledger và time series
### Data warehousing
Dùng cho:
- SQL analytics
- **Business Intelligence (BI)**
- Kiểu **OLAP**

Dịch vụ:
- **Redshift**
- **Athena**
- **EMR**

### Graph databases
Dùng để:
- Thể hiện **relationships** giữa các dataset

Dịch vụ:
- **Amazon Neptune**

### Ledger databases
Dùng để:
- Có một **list** và **ledger of transactions**

Dịch vụ:
- **Amazon QLDB (Quantum Ledger Database)**

### Time series databases
Dùng cho:
- Dữ liệu theo thời gian

Dịch vụ:
- **Amazon Timestream**

## 📊 Bảng tóm tắt
| Tiêu chí | Mô tả |
|----------|------|
| SQL / OLTP / joins | Chọn **RDS** hoặc **Aurora** |
| Tính linh hoạt | **NoSQL** thường linh hoạt hơn |
| Không cần join | Phù hợp với nhiều loại **NoSQL** |
| Lưu big objects / backups / archives | **S3** hoặc **Glacier** |
| Analytics / BI / OLAP | **Redshift**, **Athena**, **EMR** |
| Free text / unstructured search | **OpenSearch** |
| Quan hệ giữa dữ liệu | **Amazon Neptune** |
| Ledger of transactions | **Amazon QLDB** |
| Dữ liệu theo thời gian | **Amazon Timestream** |
| Cloud Native database | **Aurora** được nhắc đến như một lựa chọn |

## 💡 Mẹo ghi nhớ cho kỳ thi AWS
- Thấy **SQL + joins + transactions** thì nghĩ ngay đến **RDS / Aurora**.
- Thấy **flexibility** nhưng không nhắc nhiều đến **joins** thì thường nghiêng về **NoSQL**.
- Thấy **big objects**, **backups**, **archives** thì nhớ **S3 / Glacier**.
- Thấy **analytics**, **BI**, **OLAP** thì nhớ **Redshift / Athena / EMR**.
- Thấy **free text search** hoặc **unstructured search** thì nghĩ **OpenSearch**.
- Thấy **relationships** giữa dữ liệu thì nghĩ **Neptune**.
- Thấy **transaction ledger** thì nghĩ **QLDB**.
- Thấy **time series** thì nhớ **Timestream**.
