# 243. Keyspaces (for Apache Cassandra)

## 🎯 Giới thiệu
- **Amazon Keyspaces** là dịch vụ **managed Apache Cassandra** trên AWS.
- **Cassandra** là một **open-source NoSQL distributed database**.
- Với Keyspaces, AWS sẽ quản lý Cassandra trực tiếp trên cloud, theo mô hình **serverless**.
- Dịch vụ này **scalable**, **highly available**, và **fully managed**.

## 1. Đặc điểm chính của Keyspaces
- Tự động **scale tables up and down** theo traffic của application.
- Data của tables được **replicated 3 times** trên **multiple AZ**.
- Khi query Keyspaces, sử dụng **Cassandra Query Language (CQL)**.
- Cho phép đạt **single-digit millisecond latency** ở mọi quy mô.
- Có thể xử lý **thousands of requests per second**.

## 2. Capacity modes và tính năng bảo vệ dữ liệu
- Có 2 capacity modes, tương tự như **DynamoDB**:
  - **On-demand mode**
  - **Provisioned mode with auto-scaling**
- Tính năng đi kèm:
  - **Encryption**
  - **Backup**
  - **Point-In-Time Recovery (PITR) up to 35 days**

## 3. Use cases và mẹo thi AWS
- Use cases điển hình:
  - Lưu **IoT device information**
  - Lưu **time-series data**
- Góc nhìn ôn thi:
  - Hễ thấy **Apache Cassandra** thì nghĩ ngay đến **Amazon Keyspaces**.

## 💡 Mẹo ghi nhớ cho kỳ thi AWS
- Thấy **Apache Cassandra** → nghĩ ngay **Amazon Keyspaces**.
- Nhớ 3 ý quan trọng:
  - **Serverless managed Cassandra**
  - **CQL**
  - **PITR up to 35 days**
- So với **DynamoDB**, Keyspaces cũng có **on-demand** và **provisioned with auto-scaling**.

## ✅ Kết luận
- **Amazon Keyspaces** là lựa chọn AWS cho nhu cầu dùng **Apache Cassandra** nhưng muốn được AWS quản lý hoàn toàn.
- Điểm cần nhớ nhất: **serverless**, **CQL**, **multi-AZ replication**, **low latency**, và **capacity modes giống DynamoDB**.
